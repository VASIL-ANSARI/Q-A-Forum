I suspect the answer is in the order of execution. clears out previous DOM elements is very likely to influence the effective scroll position.

Rather than removing and then adding nodes, consider applying a CSS visibility:hidden to all current images, and then swapping them out for the newly loaded images. If there are fewer new than old, remove the stragglers at the end of the function call; if there are more new than old, then continue to add them beyond replacement.

This approach would result in minimal scroll position change.
