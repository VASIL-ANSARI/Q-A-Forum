I think this will point you in the right direction:

https://www.parse.com/questions/maximum-length-for-string-type-in-parse-data-database

There's no explicit limit on string length, but Parse Objects are limited to 128k. Otherwise, you should use a Parse File.

If you are planning to have many articles and there will be a lot of data to download you probably want to store the full text separate from the PFObject that represents the article. For example, you may have a PFObject with title, date, etc, but store the full text in another PFObject or PFFile. You can then fetch the object containing the full text when a user selects an item in your tableview.
