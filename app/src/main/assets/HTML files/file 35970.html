If you are writing to System.Diagnostics.Trace, you will need to add the following to your web.config to have it write the output to the page / trace.axd:

<system.diagnostics>
    <trace>
        <listeners>
            <add name="WebPageTraceListener"
                 type="System.Web.WebPageTraceListener, System.Web, Version=2.0.3600.0, Culture=neutral, PublicKeyToken=b03f5f7f11d50a3a"/>
        </listeners>
    </trace>
</system.diagnostics>

With thanks to: http://msdn.microsoft.com/en-us/library/b0ectfxd%28v=vs.90%29.aspx

This is particularly useful when writing to trace from business logic layers or other objects outside the scope of the web project.
