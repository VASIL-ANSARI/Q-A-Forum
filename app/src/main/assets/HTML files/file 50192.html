As Eric Lippert himself says here, this is "perfectly acceptable practice; the C# language has been carefully designed so that this is legal and practical."

Furthermore, in this case it seems unambiguous what your String property means: you have a class which is like a string, but serializable, and this property returns the underlying string. Suppose in fact that String was not a primitive type, and the primitive type was str or something. Then it does not seem to me that there would be any problem or ambiguity in calling this property (the central property of your class) simply "String" (with return type str in my hypothetical case)
