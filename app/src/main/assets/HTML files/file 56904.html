ng-include would not work without a web-server to serve your application, as some browsers would restrict the template from loading due to Cross Origin issues or due to the file:// protocol.

As per documentation:

In addition, the browser's Same Origin Policy and Cross-Origin Resource Sharing (CORS) policy may further restrict whether the template is successfully loaded. For example, ngInclude won't work for cross-domain requests on all browsers and for file:// access on some browsers.

Also, look at : Is it possible to use ng-include without web server?

TL;DR - ng-include would not work without a web server.

P.S: as mentioned in the link, there is a workaround, you can include the template in the same html file by using:

<script type="text/ng-template" id="sample.html">
  <div>This is my sample template</div>
</script>
