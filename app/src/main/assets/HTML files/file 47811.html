You are observing a symptom. And it is not the problem with quotes. In the trace a string parameter is shown with doulbe quotes. i.e.

value="20150621235959"

means the client passed a string '20150621235959' and

value=""

means the client passed an empty string ''. Which leads the the ORA-12801

The root cause is that the client is able to pass an empty string to the database.

The predicates 

action_date BETWEEN date1 and date2

returns no row if one or both of the dates is NULL. If you don't trust simple run this query

 -- return nothing
 SELECT * FROM dual where sysdate between to_date('','ddmmyyyy') and to_date('','ddmmyyyy');

There is no point why the client should send empty strings to the database, if you know, that nothing will be returned.

So IMHO there should be some validation in the client enforcing only valid strings (at least the proper lenght) are accepted. This will resolve the problem.
