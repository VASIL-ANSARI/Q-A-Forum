Consider reading documentation.

ExecuteSqlCommandAsync

does not return the selected value in the SP as return value. It returns an error code - of the SP. Which you never set. And which is an int.

You need to open a data reader or use a scalar version of execute (which returns row 1 column 1) to get the GUID.

The documentation is obviously clear on that:

Return Value Type: System.Threading.Tasks.Task<Int32> A task that represents the asynchronous operation. The task result contains the result returned by the database after executing the command.

which indicates you never read it.
