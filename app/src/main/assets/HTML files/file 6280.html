This is not really Java's style, I'm afraid.

The methods you would like remind me of Rails and Grails, concepts that don't work well with a static beast like Java (don't get me wrong, I love my beast).

Spring Roo mimics what rails and grails do and generates Controller methods like the ones you would like but that would mean you would have to use Spring Roo.

Another Idea would be to post-process your generated code by using a source code parser. But that's also going to be a tough call.

Probably your best bet is to get the code of the hbm2java tool and add a hook for your routines.
