Client side caching (browser didn't think it needed a new file), server side caching (server didn't think you needed a new file, you'd probably know about this one), or the server is in a different time zone than you (the date change time is x hours later than your local midnight).

You have to tell asp.net to discourage caching to get a page to regenerate, but that will make for worse performance, sometimes dramatically worse. Consider using javascript or donut hole caching.
