The task is maybe not really complex as long as you can always omit specifying parameter with default value (e.g. Nothing). The parameter that we can omit should be either the last one or the previous to omitted. So, what you've described is relatively easy fits in:

class Stats
{
    public:
      // some stat methods and data
};


enum Item
{
   Nothing,
   Something1,
   Something2,
   Something3
};


class A
{
   explicit A(Item item = Nothing);
   A(const Stats& stats, Item item = Nothing);
};


A::A(Item item)
{
   // in both constructors just recognize if the item is Nothing
   if (item == Nothing)
   {
      // default, no item
   }
   else
   {
      // deal with an item
   }
}

So, your client code will be able to do:

// we imply item != Nothing here
A one;                 // default
A two(item);           // with an item
A three(stats);        // with stats
A four(stats, item);   // with stats and an item

Of course you should specify an item and stats somehow.
