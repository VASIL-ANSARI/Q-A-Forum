If you are not constrained to do this within sql server, your best bet is to

  1. export your data to a text file
  2. write a program in your favorite language to process that text file (i would recommend just exporting recnum and the new groupid, and only those where the groupid is not null)
    • even over 35 million+ rows, this should only take a few minutes
  3. use a bulk insert to bring your data back into sql server
    • again, just a few minutes

  4. use an UPDATE ... FROM statement to join your bulk-inserted table back to your original table to update the groupid column

    UPDATE o 
SET GroupID = bi.GroupID
FROM OriginalTable o 
    INNER JOIN BulkInsertTable bi 
    ON o.RecNum = bi.RecNum
    • this one will take significantly longer, but since the time it is taking is the time to write your data to disk, it is unavoidable
