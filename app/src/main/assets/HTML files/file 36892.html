Well first of all, as it's said in documentation passing a code to setTimeout is strongly discouraged:

code: in the alternate syntax is a string of code you want to execute after delay milliseconds (using this syntax is not recommended for the same reasons as using eval())

So I would recommend calling functions using following syntax:

setTimeout(checkAds, 25);

But, if you still want to use it like you do, you need to define these functions in window context, see jsfiddle:

window.checkAds = function() {
...
}

It wasn't working because when interpreter dynamically wrapped your code in anonymous function, your functions were be undefined in new context.
