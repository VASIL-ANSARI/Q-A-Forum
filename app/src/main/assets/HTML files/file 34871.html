You have a very poor data layout. You should have a third table, OrderItems that has one row per order and item in the order.

You can do what you want using string manipulations. Unfortunately, you cannot take advantage of indexes and other database optimizations.

select i.item_code, count(*) as cnt
from orders o join
     items i
     on concat(' ', o.item_code, ' ') like concat('% ', i.item_code, ' %')
group by i.item_code
order by count(*) desc;

EDIT:

You can create the OrderItems table by running a query quite similar to the one above:

select o.order_id, i.item_code
from orders o join
     items i
     on concat(' ', o.item_code, ' ') like concat('% ', i.item_code, ' %');

Then insert this into the new table or use create table as.
