Are you sure your logback.xml file is in the correct location and is being detected?
Try intentionally putting in errors in the config file, and seeing if you get messages in the console (Logback will do this automatically).

IF not you can see the Logback configuration how to pass or run your application like below.

java -Dlogback.configurationFile=/path/to/logback.xml YourMainClass

Let us begin by discussing the initialization steps that logback follows to try to configure itself:

  1. Logback tries to find a file called logback.groovy in the classpath.
  2. If no such file is found, logback tries to find a file called logback-test.xml in the classpath.
  3. If no such file is found, it checks for the file logback.xml in the classpath.
