I can't say for certain, but I think it ought to be fine. Generally Rails is pretty proactive about telling you when you're trying to use a name that'll conflict, so why not try it out and see what happens?

If the table name (as opposed to the model name) turns out to be an issue, you can always use self.table_name = "some_other_table_name" in your model (as described in Active Record Basics) to give it the name of another table to use.
