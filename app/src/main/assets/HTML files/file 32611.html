I would suggest a simple approach first using an external service like firechat (which is built on top of firebase). You can build your own chat really easy, worrying only about the client part. The problem is that it's free only for a limited amount of users.

If you want something 100% free you can use the ejabberd. This has to be installed on top of XAMPP (for development environment) and is built with the JABBER/XMPP protocol (standar messaging protocol). This represents the server part of a chat application, now you need to worry about the client. You can test your server with any Chat app that is built for the same protocol (like iChat). To build your own chat client, use this XMPP framework for iOS. Everything is well documented.

One additional note. The second approach is MUCH MUCH more complicated (it was for me at least).

Hope this helps.
