Setting the <welcome-file> in web.xml means the webapp will always start on this page, regardless of authorized state.

The typical authorization sequence in a app is:

  1. Serve index.html (or .jsp), the welcome page.
  2. User tries to access a protected resource, say /protected.jsp
  3. Server answers 401 Authorization Required with a redirect to /login.jsp
  4. User submits login for.
  5. Server sets session cookies indicating authorized state and redirects back to /protected.jsp
  6. User views /protected.jsp

If you want the user to start on the login page, you are probably best off setting up the above flow, and then do an automatic redirect on your welcome page index.html to /protected.jsp.
