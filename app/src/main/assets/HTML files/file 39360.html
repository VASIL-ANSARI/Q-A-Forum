If you check the console, you will find the error: "t3 is not defined". This comes because you never defined t3. If I put in the following near the top of your function, it works (up to a point):

var t3 = document.getElementById("Text1").value;

It will eventually break, since you have more problems than that. For one thing, a big reason why your array isn't printing is because you never defined its contents.

Please note that it is a hell for anyone to read code where the variable t1 is contents of element Text2, and t3 content of Text2, and where a loop counters are a, i and n. Consider changing variable names to className, methodCount, variableCount, etc. It is hell for me right now, and I guarantee it will be hell for you too in about a week, if it isn't already.

Also, be aware that document.write is a very dangerous and destructive force. Use it only when it is absolutely necessary, and even then think thrice. Use element.innerHTML = ..., or element.appendChild(...) with document.createElement(...). The first method is easier, and it is almost always a better idea and more controllable than document.write. I see you are already using innerHTML, so this should not be hard. Treat document.write as if it is radioactive: if you absolutely have to touch it, be smart, and wear a biohazard suit.

Finally, your loops do not do what you want, not even close. As I said, it was hard to even follow your logic there due to nonsensical variable names and weird limits. You will find that when you give your variables better names, you will get clarity to see what your program is actually doing.
