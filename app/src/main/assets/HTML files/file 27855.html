Depends on your needs but perhaps it is sufficient to alter the launchmode of your activity in manifest:

android:launchMode="singleTask"

In contrast, "singleTask" and "singleInstance" activities can only begin a task. They are always at the root of the activity stack. Moreover, the device can hold only one instance of the activity at a time â only one such task.

This would prevent onCreate being called before onDestroy.

Edit:

Another possible solution would be to manually check if there are other processes running your application in e.g. onCreate:

    ActivityManager activityManager = (ActivityManager) this
            .getSystemService(ACTIVITY_SERVICE);
    List<RunningAppProcessInfo> procInfos = activityManager
            .getRunningAppProcesses();
    int counter = 0;
    for (int i = 0; i < procInfos.size(); i++) {
        if (procInfos.get(i).processName.equals(this.getPackageName())) {
            counter++;
        }
        if (counter > 1) {
            // send broadcast to make other activities call finish() or the like
        }
    }

Hesitated with posting this answer as it feels abit hackish but since no other proposals have been posted until now you might as well give it a try.

I used it for a different scenario where the phone would sometimes start my app twice after boot. With this code I could prevent the second start to complete by simply calling finish.
