The problem is the test target that comes with the library. In C and C++ you usual have a dedicated command line executable that runs a test suite. On OSX there is target type for that called Command Line Tool. Xcode references target types with a reverse DNS identifier, in this case com.apple.product-type.tool. On iOS you do not have a target type for command line tools because, well, there is no public command line that you could run the executable from.

That being said, you have already found the solution yourself: deactivate the test target generation in CMake for that library if you want to build for iOS.
