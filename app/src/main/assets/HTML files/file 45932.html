The MultipleInputs class is contained in the hadoop-mapreduce-client-core artifact. Your project has a provided dependency on hadoop-client. The hadoop-client artifact is a thin wrapper that brings in multiple other Hadoop artifacts via transitive dependency.

https://github.com/apache/hadoop/blob/release-2.6.0/hadoop-client/pom.xml

Based on this, I recommend making the following changes:

  1. Remove <excludeTransitive>true</excludeTransitive>. The hadoop-client dependency alone is just a pom.xml, and transitive dependency resolution is required to reach hadoop-mapreduce-client-core and other Hadoop artifacts.
  2. Change <includeScope>runtime</includeScope> to <includeScope>provided</includeScope> to match the dependency scope used in the <dependency> section.

The maven-dependency-plugin configuration is copying dependencies to the target/lib directory. I assume your IntelliJ project is already set up to add all jars from this directory to its classpath.
