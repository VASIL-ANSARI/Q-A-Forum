Don't use COM ports. Yuk! :-)

Instead use Sockets or a Bluetooth API, then you can accept lots and lots of connections to the same Bluetooth 'server'. For instance my library 32feet.NET.

See e.g. 32feet.NET docs -- Bluetooth%20Server-side

However. If you mean one client device (eg Android here) connecting to one server device then there is a restriction. With RFComm, as used by SPP, there can be only one connection from one device to the same server application. To get two connections between two device then there will need to be two RFComm/SPP servers running.
