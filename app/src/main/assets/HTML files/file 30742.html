in your query

select col from tab t1 where not exists(select col from tab t2 where t2.col < t1.col)

the 

select col from tab t1

selects all entries then the subquery in

where not exists(select col from tab t2 where t2.col < t1.col)

selects every entry that is smaller than the current entry it wants to comapre with. So for the entry 1 in the first query it takes no entry since there is no entry that is smaller than 1. then the 

not exists()

kicks in and since there is no result from the subquery, the not exists is true and the whole query returns the 1.

Any other value will not be returned since for any other value you will get a result in the subquery thus leaving the not exists() with the value false
