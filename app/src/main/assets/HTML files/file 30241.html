Try setting a break for System.Web.HttpException (there is a Find... button in the exceptions dialog http://msdn.microsoft.com/en-us/library/h22dk1y6.aspx). After applying this setting the debugger should stop when the exception is thrown (inside System.Web.UI.WebControls.DropDownList.VerifyMultiSelect method). In the watch window you should be able to evaluate this and check to which control it refers.
