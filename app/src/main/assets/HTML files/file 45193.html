As this is HTML email, just use nested tables. It's ugly and unsemantic, but it's the standard way of coding for email, because of the lack of support for the box model.

<table>
  <tr>
    <td>
      ... DESCRIPTION 1 CONTENT ...
    </td>
  </tr>
  <tr>
    <td>
      <table>
        <tr>
          <td>... TABLE 1 CONTENT ...</td>
        </tr>
      </table>
    </td>
  </tr>
  <tr>
    <td>
      ... DESCRIPTION 2 CONTENT ...
    </td>
  </tr>
  <tr>
    <td>
      <table>
        <tr>
          <td>... TABLE 2 CONTENT ...</td>
        </tr>
      </table>
    </td>
  </tr>
</table>

Mailchimp has loads of free templates that should give you a headstart on layouts.
