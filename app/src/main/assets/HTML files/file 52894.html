If the sorting is a business requirement, I'd write a test for the method producing (and possibly displaying) the sorted results. Pass it an unordered collection of GroceryItems and verify the order is correct.

Putting a test only on the enum is probably the wrong way to go (though you can do both tests). The reason I say that, is that you're not guaranteeing your code actually sorts by department.

These would all go undetected:

groceryItems.OrderBy(g => g.Name).ThenBy(g => g.Department);

groceryItems.OrderByDescending(g => g.Department).ThenBy(g => g.Department);

groceryItems.OrderBy(g => g.Name);

groceryItems.OrderBy(g => g.Department);

Even though the enum is correctly sorted.
