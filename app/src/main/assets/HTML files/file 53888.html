The problem with your query is the field UserInfo.Id, in the SELECT clause. Because it is not part of the GROUP BY SQL Server doesn't know what to do with it.

When using GROUP BY any field that does not appear in the grouping must be aggregated, using one of the aggregate functions.

Consider this example table:

Country Sales
UK      10.10
UK      20.10
USA     15.25

Grouping by country would return:

Country
UK
USA

Adding sales to the SELECT causes a problem. There are three sales values and only two rows. SQL Server needs instruction on how to turn the three source rows into 2 output rows. You could SUM the sales. That would return:

Count   Sales
UK      30.20
USA     15.25

Or you could always add Sales to the group by. That would return the original three rows.

As a rule of thumb, when using GROUP BY any field not included in the grouping must be wrapped in a aggregate function (SUM, MIN, MAX, etc).
