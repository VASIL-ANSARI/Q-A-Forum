We had the same issue (on devices running iOS 4.x only btw..) The cause of it has been the use of mutable objects in the cache. We put a NSMutableDictionary in the cache and and manipulated it afterwards. Then when the cache was released (to be deallocated) something seems to have happened that later crashed the app on suspend.

Hope this also solves your issue!

Update:

We found a solution thanks to this dev forum entry. It appears that NSCache prior to iOS 4.3 was not unregistering itself from the notification center.

We worked around this issue by creating a simple subclass of NSCache and implement the -dealloc method like so (ARC version):

- (void)dealloc;
{
    // NSCache had an issue on iOS prior to 4.3 where it didn't unregister from notifications and caused crashes
    // see https://devforums.apple.com/message/421845#421845
    [[NSNotificationCenter defaultCenter] removeObserver:self];
}

Now use this class instead of NSCache and you should be okay. Fingers crossed. :)
