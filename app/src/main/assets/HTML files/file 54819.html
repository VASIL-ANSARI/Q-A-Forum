The basic principle of unit testing is that each unit (Car::Wheel, Car::Door) should be tested separately. When you compose two classes you can be sure, that the underlying elements are properly implemented.

Instead of having run method you should have methods with single responsibility (just build and not build & serialize):

  def self.build(wheels, doors)
    car = Car.new
    car.wheels = Car::Wheel.load_json(wheels)
    car.doors = Car::Door.load_json(doors)
    car
  end

  def self.run(wheels, doors)
    self.build.to_s
  end

This way you can easily test output of build method as it return an object.
