This could be easily done with some generic helper functions.

For example, instead of manually providing the raw HTML for each value you could write a function that clears and populates the <select> element according to a single value that is passed.

In preparation, you'll want to create an object containing the values of the first <select> as indexes to the list of second <select>. Something like this - 

var beverages = {
 'wine' : ['Red','White'],
 'beer' : ['dark','pale'],
 'juice': ['mango','grapefruit']
};

Now we know that beverages['beer'][0] will get us a cold bottle of beer.

From here we can write a more generic function to help us with the HTML manipulations...

function populateSelect(category){
  // remove all existing options.
  $("#secondSelection").find('option').remove(); 
  $.each(beverages[category],function(index,elem){
    // for each item in beverages[category],
    // we add an option to the <select>.
     $("#secondSelection").append('<option>'+elem+'</option>');
  });
}

You would call this function like this -

populateSelect($("#firstSelection").val());

The markup you'd have to have would be something like this - 

<select id="firstSelection">
  <option value='beer'>Beer</option>
  <option value='wine'>Wine</option>
  <option value='juice'>Juice</option>
</select>
<select id="secondSelection">
  <option>Choose a Category</option>
</select>

For more complex structures like the ones you described, you could just nest more options in the original object and it's arrays...
