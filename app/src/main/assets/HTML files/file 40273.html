I'd certainly expect a bad access if you were not using ARC, in which case a dangling pointer is extremely likely.

Regardless, I would make sure that didEndElement sets element variable to nil at the end of the method (or else you may end up with unintended extra characters in title and link values). This also minimizes the incidental and unnecessary references to the elementName you received in didStartElement beyond the point that you actually needed it.

Personally, if I were going to use the elementName passed to me in didStartElement beyond the scope of that method, I'd be inclined to either copy the value passed to me or use some other state variable to keep track of whether I was parsing title or link or whatever.
