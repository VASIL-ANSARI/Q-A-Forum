Since you're refactoring, you should think about getting rid of null and use Java 8's Optional. Guava's Optional wouldn't be any good here because its transform method doesn't allow a Function to return null (and tbh, I don't know why they chose that instead of doing like Java 8 chose to do).

Then with Java 8, it's rather simple to use:

Optional<Object> object = ...;
Optional<Something> something = object
    .map(obj -> obj.getNextObject())
    .map(nextObj -> nextObj.getSomething());

// Now use this instead of checking for " != null ":
if (something.isPresent()) {
  doSomething(something.get());
} else {
  doSomethingElse();
}

If you're not using Java 8, you can still try to implement something similar yourself, this pattern isn't really hard to implement by yourself once you have a clear API.
