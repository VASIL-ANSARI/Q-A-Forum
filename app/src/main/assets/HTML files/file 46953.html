You can run a native sql query with doctrine to achieve your objective. See an example here: http://doctrine-orm.readthedocs.org/en/latest/reference/native-sql.html

The second option would be to use bidirectional associations. If performance is your concern you can enable the LAZY or EXTRA_LAZY fetch modes. Thus the performance will be minimally affected if at all.

More on extra lazy associations: http://doctrine-orm.readthedocs.org/en/latest/tutorials/extra-lazy-associations.html

In doctrine associations are considered lazy by default. This means that the entities will be fully loaded only if used (ie. property access, search, etc).

The extra lazy fetch mode diminishes even more the number of method which trigger a full entity hydration.
