You have to instantiate a hardware button object then listen for the 'backpressed' event. There is a property on the event 'handled' that you can set to alert the system that you've captured the back button press and don't want the system to do anything else.

The documentation for this event can be found here: http://msdn.microsoft.com/en-us/library/windows.phone.ui.input.hardwarebuttons.backpressed.aspx

var hardwareButtons = Windows.Phone.UI.Input.HardwareButtons;
hardwareButtons.addEventListener("backpressed", function (e) {
   // Navigate back in your webview. 
   e.handled = true; // Notifies OS that you've handled the back button event.
});

Hope this helps.
