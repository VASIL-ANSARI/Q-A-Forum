If your XML generator doesn't generate an Extended-Attribute tag for attributes that has no value, so there is no way to reference them in your XSLT.

There should be a way to tell XSLT processor about what properties (how many Extended-Attribute tag) should exists in Extended-Attributes collections. This way maybe you can have a XSLT code which looks for those attribute names in the collection and if it couldn't find them, use the default empty apostrophes value.
