According to the docs, you need to inherit your RegisterUser class from AbstractBaseUser class. You also need to specify some key implementations.

Quoting the docs

The easiest way to construct a compliant custom User model is to inherit from AbstractBaseUser. AbstractBaseUser provides the core implementation of a User model, including hashed passwords and tokenized password resets.

Other key implementations you need to specify in your custom user model are

  • USERNAME_FIELD

  • REQUIRED_FIELDS

  • is_active

  • get_full_name()

  • get_short_name()
