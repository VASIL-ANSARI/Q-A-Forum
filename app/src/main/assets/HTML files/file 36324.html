<ListBox.ItemTemplate>
     <Label Content="{Binding Path={Binding CustomerPath}}"/>
     <Button Width="20" Height="20" FontWeight="Bold" Content="Ã"/>
 </ListBox.ItemTemplate>

Binding is already wrong. If your ListBox's ItemsSource is Customers and you want to bind to it's content then just use {Binding Path=CustomerPath}. The answer to your question is yes, you can specify at runtime the path for a data-bound value. You'll have to load each Template for the ListBox then set the bindings at runtime.

Pseudo code:

  1. Load item template
  2. Get the label control
  3. Set the binding to what the caller sets the property to bind it to

How can I instruct the ListBox to be able to work with an unknown data type, but be able to choose the path for the data-bound value?

Binding will call ToString() on the class that it is bound to. So if your binding was correct and assuming you have Customer as object

<Label Content="{Binding .}"/>

If you did not override the ToString() of the Customer object it'll show the default ToString() but if you do then that'll be the one that will be shown in the Label. This is an ugly solution just to substitute it for your DisplayMemberPath that was given by the ListBox

Ideally, I would do the setting the binding at runtime rather than creating an interface not just because of what you said but succeeding objects that will be used for this control will have to implement that interface.
