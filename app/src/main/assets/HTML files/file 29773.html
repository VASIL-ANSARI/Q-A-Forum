I would suggest you look at the OWASP-ESAPI project, but the basic principles are as follows

  • never trust input from the user
    • lets say you need a number, then you would validate that number to ensure it is a number, otherwise reject it, this ensures that only numbers come through
    • if you need a title say, then you can already restrict to [A-Za-z]{2,4} for example.
  • always encode output
    • anything that comes from the database/user should be encoded properly
