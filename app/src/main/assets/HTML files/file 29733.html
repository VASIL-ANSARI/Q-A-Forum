The Spring MVC documentation explains this in the Supported method return types section

The following are the supported return types:

[...]

  • A String value that is interpreted as the logical view name, with the model implicitly determined through command objects and @ModelAttribute annotated reference data accessor methods.

Your method returns a String, so its value is therefore interpreted as a view name. You probably have declared an InternalResourceViewResolver in your context which takes the view name and tries to resolve an InternalResourceView. This is done by using HttpServletRequest#getRequestDispatcher() with the view name (prefixed and suffixed as configured).

Read the documentation on resolving views.
