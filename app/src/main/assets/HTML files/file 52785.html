This looks like a homework question, but since I already mostly answered it in my comment, I'll just go ahead and move it down here to an actual answer and add some explanation.

Your query: 

UPDATE `oc_product` p 
LEFT JOIN 'oc_product_description' d 
ON p.product_id = d.product_id 
SET p.image = "/catalog/tonerthumb.gif" 
WHERE d.description like '%toner%'

The primary problem you have is that you are using single quotes in an attempt to delimit an object. Back-ticks are used for that. Single quotes are using as a string delimiter. In my comment above, I mentioned that double-quotes may not be correct for a string delimiter, but after looking it up, you can actually use single or double quotes interchangeably in mysql, so you can ignore that bit. The only real problem you have is the single-quote vs back-tick.

In order for your query to be correctly formed, you just need to replace the single-quotes with back-ticks on the join clause of your query:

UPDATE `oc_product` p 
LEFT JOIN `oc_product_description` d 
ON p.product_id = d.product_id 
SET p.image = "/catalog/tonerthumb.gif" 
WHERE d.description like '%toner%'

Although the double quotes are fine to sue for string delimiters, as I mentioned, it is better to be consistent when writing queries, so I would go ahead and change the query just to bring some consistency:

UPDATE `oc_product` p 
LEFT JOIN `oc_product_description` d 
ON p.product_id = d.product_id 
SET p.image = '/catalog/tonerthumb.gif' 
WHERE d.description like '%toner%';

I would also get into the habit of ending all of your queries explicitly with a semi-colon.
