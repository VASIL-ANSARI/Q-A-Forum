Yes, but it quickly gets ugly coding it. However, pick up the package ncurses and all will become clear.

Basically, the trick is that you must set the terminal device to "raw" mode so you can get the special key sequences directly in your program, then have to interpret them -- they tend to be things like "<ESC>33;5A". Then you need to emit the control sequences that move you around on the screen, and all of these things are device dependent so xterm may differ from gterm and so on.

ncurses(3) already hides all that stuff.
