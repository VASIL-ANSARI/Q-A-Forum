You should call

ui->nameEdit->setText(texto); //here it is the problem
ui->diagEdit->setText(names);
ui->dateEdit->setDate(fechas);

in the first window (ventana), but now you're calling it inside pacientes. So you have to send the path to the file from the pacientes to ventana and set the text in ventana.

In pacientes you should have something like this:

void pacientes::on_pacienteAceptar_clicked()
{
    QString pathToDirectory = "C://Users//Jaime//Desktop//interfaz//pacientes";
    QString currentFileName = ui->pacienteLista->currentText();
    QString pathToFile = QString("%1//%2").arg(pathToDirectory, currentFileName);
    emit customSignalSetText( pathToFile );
}

In ventana you make a custom slot that does all the work:

void ventana::customSlotSetText( QString const & text )
{
    if(!text.isEmpty())
    {
        QFile file(text);
        ...
    }
}

And in your application where you have pacientes and ventana, you connect your custom signal from pacientes to your custom slot in ventana.

EDIT: This signal-slot connect must happen in a place where you have access to an instance of pacientes and an instance of ventana.

EDIT2: The connect must also happen within a QObject.

int main(...)
{
    ...
    ventana1 w;
    w.show();
    pacientes pacientesObj(w);
    ...
}

In the constructor of pacientes:

pacientes::pacientes( ventana1 receiver, QWidget* parent )
{
    ...
    connect( this, SIGNAL( customSignalSetText( QString ) ), &receiver, SLOT( customSlotSetText( QString ) ) );
}

Note that you can not connect signals and slots between classes, only between instances of those classes.

As for emitting the signal, look here. It is probably one of the following: (1) you did not use the Q_OBJECT macro or (2) you did not declare the signal correctly (with signals:). If it was the first, you'll have to run qmake again.
