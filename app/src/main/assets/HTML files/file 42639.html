as you can see in this code:

CELERYBEAT_SCHEDULE = {
    'add-every-30-seconds': {
        'task': 'tasks.add',
        'schedule': crontab(hour=7, minute=30, day_of_week=1),
        'args': (16, 16)
    },
}

the celery beat is actually the process itself,think of it like a function with settings. the function is the task and the settings are the arguments,name and schedule, here comes the crontab, its a period of a time that tells celery when to cycle through it.
