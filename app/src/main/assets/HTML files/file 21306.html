extern is exactly what you want in the header file, but it needs to be lowercase E. Your terminology isn't quite right - your header file needs a declaration whilst exactly one (thanks to the "One definition rule") .cpp file wants a definition.

E.g. in the header file:

//globals.h
extern cGameState *currentState; // declare currentState

and in the .cpp file:

// this makes the compiler check the types match
#include "globals.h" 
//globals.cpp
//Game state object
cGameState *currentState = NULL;

Personally though I'd look to avoid having lots of global state in your application.
