Windows has restrictions on the characters that can be contained in a file name. From Naming Files, Paths, and Namespaces:

Use any character in the current code page for a name, including Unicode characters and characters in the extended character set (128â255), except for the following:

The following reserved characters:

  • < (less than)
  • > (greater than)
  • : (colon)
  • " (double quote)
  • / (forward slash)
  • \ (backslash)
  • | (vertical bar or pipe)
  • ? (question mark)

  • * (asterisk)

  • Integer value zero, sometimes referred to as the ASCII NUL character.

  • Characters whose integer representations are in the range from 1 through 31, except for alternate data streams where these characters are allowed. For more information about file streams, see File Streams.

If any of your file names contains one of these characters, it will be unreadable on a Windows system.
