I found a way to resolve the issue. My pickle file was being created by a cronjob from root user. Python program was developed in virtual environment. Global environment does not have pandas. So when the root user ran the cronjob, it created pickle file successfully but something was wrong with this file. I modified the cronjob to use python binary from my virtualenv and that that fixed the issue. I can see the size difference in pickle file created by global python and virtualenv python.

I am still not sure how root user was able to run the python file while it does not have pandas available.
