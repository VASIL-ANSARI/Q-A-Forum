I think you might be overcomplicating things a little bit.

From what I understand, you need a symbolic link from your docroot to the data folder.

Create one using the following command:

mkdir -p /data1/home/spaceweather/htdoc/cases #just in case it does not exist
ln -s  /data1/public/cases /data1/home/spaceweather/htdoc/cases/data

This will create a symbolic link called "data" which will point to your /data1/public/cases directory.

If you ls -l in that directory you should see something like

data -> /data1/public/cases

Try to cd into the data directory to verify this worked.

This shouldn't be necessary for you as you have it already in your .htaccess, but leaving it here for anyone else anyway:

If this doesn't work through the web browser at first, you may need to add the FollowSymLinks to your htaccess: Options +FollowSymLinks. This tells Apache that it is OK to follow symbolic links such as the one you created.

Hope this helps. If I misunderstood your situation or requirements, please say so and I will edit appropriately.
