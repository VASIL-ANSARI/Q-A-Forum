In the real world I probably wouldn'd do this in navigational elements as hx tags are meant to be used to denote section headings (h2 - h6) or page headings (h1). I've also refactored your tables to lists. There is more flexability with design this way.

Javascript/Jquery

function switchContent(activeSelector){
    $(".content").not(activeSelector).fadeOut(500, function(){

        $(activeSelector).fadeIn(500)});
}

$( document ).ready(function() {

   $("#mResumeLink").click(function () {
       switchContent("#mResumeContent");
   });

   $("#mPhotographyLink").click(function () {
       switchContent("#mPhotoContent");
   });


   $("#mPrintLink").click(function () {
       switchContent("#mPrintContent");
   });


   $("#mWebLink").click(function () {
       switchContent("#mWebContent");
   });
 });/*End of $(document).ready()*/

Note that I've used a callback function on the fadeOut. This function will execute once the fade out is complete.

This isn't a complete solution but should get you headed in the right direction
