I don't think current mysql (5.7 VERSION) including such spatial functions (maybe in the future) (http://dev.mysql.com/doc/refman/5.7/en/functions-that-create-new-geometries-from-existing-ones.html#functions-that-produce-new-geometries)

There is line: OpenGIS proposes a number of other functions that can produce geometries. They are designed to implement spatial operators. These functions are not implemented in MySQL.

Therefore, since you can't do it in the MySQL, the only way to make it happen is on the application side, whatever you are using. For instance, OpenLayer javascript can do this: (http://gis.stackexchange.com/questions/19964/how-to-merge-two-polygons-in-openlayers)

Or, you can use an spatial-oriented database, such as http://postgis.net/, or Oracle Spatial :)
