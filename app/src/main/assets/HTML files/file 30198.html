Yes you are right this is not following MVVM and the problem is in how you are setting ItemSource -

ItemsSource = "BaseClassesData",

I think you might have to first specify TreeNode's DataContext and then create a binding to specify ItemSource (instead of just specifying ItemsSource = "BaseClassesData") Or directly specify ItemSource as you mentioned in your comment.

Both the above cses will require you to access your ViewModel in View, and there is nothing wrong in View accessing ViewModel; in MVVM view always depends on ViewModel and always knows about it and uses it ( Indirectly through Bindings and Directly in code).

Finally, In case this is not just a temp. project and needs to be maintained/enhanced in future I would suggest you to implement proper MVVM; you will come across a lot of requirements like selection, expansion, dynamic loading, search etc., and doing all this in code will be very cumbersome and unmaintainable.

Please have a look at following article to do so -

Simplifying the WPF TreeView by Using the ViewModel Pattern

Also have a look at following SO question having code for doing something very similar using MVVM -

WPF: Binding TreeView in MVVM way step by step tutorial
