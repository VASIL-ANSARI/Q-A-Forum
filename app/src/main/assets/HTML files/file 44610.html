I'm no expert in the specific technologies you're using, but on first glance I'd say that the problem is that your listeners are all listening for the same, global new reply event, without identifying if the reply is relevant to their specific question id.

Unless this is something SocketIO handles itself, you probably want to either create different reply channels to listen to for each form, or check the question id in your listener.

For example, send the ID with the data:

socket.on('new_reply', function(data){
    if(data.question === id) {
        $repliesDiv.prepend("<div id='areply'>"+data.reply+"</div>");
        $('#areply').addClass("fadeInTop");
    }
});

Or make unique 'channels', like this:

socket.on('new_reply.' + id, function(data){
    $repliesDiv.prepend("<div id='areply'>"+data.reply+"</div>");
    $('#areply').addClass("fadeInTop");
});

I'm not sure if / how this last one would impact performance though.
