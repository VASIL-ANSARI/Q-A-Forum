It's a little awkward but you can use two defaultdicts, one to keep track of all the docs for each person and the other to keep a sum of all the values of the docs

import csv
import collections

name_sum = collections.defaultdict(int)
docs_for_name = collections.defaultdict(list)

with open("csv1.csv") as csvfile:
    csvreader = csv.reader(csvfile)
    for row in csvreader:
        name_sum[row[1]] += int(row[2])
        docs_for_name[row[1]].append(row[0])

with open("outputcsv.csv", "w", newline="") as csvfile:
    csvwriter = csv.writer(csvfile)
    for key in sorted(name_sum.keys()):
        row = [", ".join(sorted(docs_for_name[key])), key, str(name_sum[key])]
        print(row)
        csvwriter.writerow(row)

You didn't post an actual csv file, just what the csv file looks like in an editor like excel, so I'm assuming your csv looks like this:

doc_01,Bob,5
doc_03,Sam,1
doc_04,Bob,1

This ends up outputting a csv file like:

"doc_01, doc_04",Bob,6
doc_03,Sam,1

The first defaultdict, name_sum just keeps track of the sum for all the values for each name, so it ends up looking like

defaultdict(<class 'int'>, {'Bob': 6, 'Sam': 1})

And the other defaultdict to keep track of all the docs for the name, docs_for_name ends up looking like:

defaultdict(<class 'list'>, {'Bob': ['doc_01', 'doc_04'], 'Sam': ['doc_03']})

Then you have all the information you need to write each row in your new csv file
