There is a simple hack you can use to achieve this effect. You can encompass the ImageView inside another UIView. Give Autolayout Constraints for the ImageView's Leading, Top and Bottom Space. Don't add a trailing space constraint. Now when you change the UIViews width, you can see the image being revealed as per your input.

Now Connect up the required outlets in viewController.h. I'm using a slider for demonstration:

//Trailing space constraint of CoverView
@property (weak, nonatomic) IBOutlet NSLayoutConstraint *trailingSpaceConstraint;

@property (weak, nonatomic) IBOutlet UISlider *slider;

@property (weak, nonatomic) IBOutlet UIView *speedoHolder;

- (IBAction)sliderValueChanged:(id)sender;

And Now the implementation:

- (IBAction)sliderValueChanged:(id)sender {
  CGFloat newVal = self.slider.value;

  self.trailingSpaceConstraint.constant = self.speedoHolder.frame.size.width * (1-newVal);

  [self.view setNeedsUpdateConstraints];
  [self.view layoutIfNeeded];
}

PS: dont forget to enable Clips Subviews for CoverView in Attributes Inspector.

Update: you can fine tune the hiding by modifying the value of trailingSpaceConstraint.constant. i.e. if you have a max speed and current(GPS) speed, do this:

self.trailingSpaceConstraint.constant = self.speedoHolder.frame.size.width * (gpsSpeed / maxSpeed);
