The option is textvariable, not stringvariable.

Unrelated to this problem, when you do height = Entry(...).grid(...), height will always be None since that is what .grid(...) returns. It is a best practice to separate widget creation from widget layout.

When you do that (separating creation from layout) you don't actually need to use StringVars, since you can get the value directly from the widget itself (eg: hgtval = float(height.get())
