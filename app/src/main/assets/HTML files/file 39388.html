You'd probably need some kind of back-end to do this, or at least make sure that the time intervals are specified individually in advance.

It's likely that you can hook up the PCs to one another in some kind of peer-to-peer arrangement, perhaps using the new-fangled RTC API (see http://www.html5rocks.com/en/tutorials/webrtc/basics/#toc-rtcdatachannel).

Complete millisecond timing accuracy may be elusive, however, but if you can live with a tolerance of (say) 10-20 milliseconds, you should be ok. Depends on what else the computer is doing, of course.
