I found the problem in the line $out = 'open 192.168.0.22' . PHP_EOL; more specifically PHP_EOL . Since the server target is running Windows, it expects \r\n as a character return. My computer (localhost) is running Windows, which makes PHP_EOL = \r\n, but once I deployed the application on a Linux server, the PHP_EOL there is set to \n. My problem was solved with the following line:

$out = 'open 192.168.0.22' . "\r\n";

Note: the character \r\n must be wrapped in double quotes mark ("") for it to work.
