Why would you work this way? In a REST world, this makes no sense.

You could, for instance, make a WebApi controller method like so:

public User Get(string authToken)
{
    ...
}

Then, assuming this is a method of the User WebApi Controller, you would call it this way:

http://servername:port/api/user/get/ABC123

and then your method would get called with authToken = "ABC123".

It is a convention in ASP.NET MVC to route WebApi Controller methods using an /api prefix in the URL.

For more info, see this article on www.asp.net: http://www.asp.net/web-api/overview/getting-started-with-aspnet-web-api/tutorial-your-first-web-api and another on MSDN: http://code.msdn.microsoft.com/ASP-NET-Web-API-Tutorial-8d2588b1

Good luck!
