No, it could not throw ClassNotFoundException. With JDBC4, if the Driver isn't registered, you'll just get an exception stating that no registered driver exists for that url. Something like

Exception in thread "main" java.sql.SQLException: No suitable driver found for <your url>
    at java.sql.DriverManager.getConnection(Unknown Source)
    at java.sql.DriverManager.getConnection(Unknown Source)
    at ru.expbrain.flib.Main.main(Main.java:15)

The javadoc for DriverManager.getConnection(String) states

The DriverManager attempts to select an appropriate driver from the set of registered JDBC drivers.
