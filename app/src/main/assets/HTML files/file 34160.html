You can use id is you like just use the accessor methods instead of the property.

+(BOOL)saveGroupImageToDisk:(UIImage*)image toEntity:(id)entity withparams:(NSDictionary *)dictionary;
{
    [entity setPropertyName:nil];
    ...
}

E.g. if you have a properties declared as

@property (nonatomic, copy) NSString *name;
@property (nonatomic) NSInteger *age;

You would access them in the following way:

[object setName:@"Bob"];
NSString *name = [object name];

[object setAge:40];
NSInteger age = [object age];

All you do is lose the compile time checking that that an instance of that class has that property defined.
