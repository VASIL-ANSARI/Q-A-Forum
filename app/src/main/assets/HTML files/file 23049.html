This is asked and answered in comments 74 and 75 at https://bugzilla.mozilla.org/show_bug.cgi?id=748343 .

In short, the latest Java plugins disallow code="java.applet.Applet". And there seems to be no way to get the plugin to access a jar file in your extension. The workaround is to put a jar file at a publicly accessible HTTP site; the jar file just needs to contain some trivial applet, like public class Applet extends java.applet.Applet { }. Here is the overlay HTML copied from the Firefox bugzilla comment:

<html:div name="appletDiv">
    <html:embed id ="appletID" type="application/x-java-applet" 
        archive="https://mysite.com/applet.jar" code="Applet" 
        pluginspage="http://java.com/download/"  
        MAYSCRIPT="true" width="0" height="0" />     
</html:div>
