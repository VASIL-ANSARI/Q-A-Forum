In Rails, unsafe values are automatically escaped. & is considered as an unsafe value, so it gets escaped to &amp;.

But I think a generated link should not be escaped, should be used as is I mean. So I think you should use the raw output in here with :

 <%== tweets_show_replies_path(:parent => @parent, active: params[:active]) %>

or :

 <%= raw tweets_show_replies_path(:parent => @parent, active: params[:active]) %>
