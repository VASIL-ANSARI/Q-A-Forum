there is no link between front and rear

Of course there is - here is how it is established:

if(front==rear&&front==NULL)
{
    ptr->next=NULL;
    front=rear=ptr;
}

front points to the first inserted element. Initially, rear points to the same element as well. As you add more elements to the queue, rear moves on, while front remains pointing to the same initial element.

show() takes that element, and traverses the linked list using it.

Note that if items are always inserted using insert, the while loop is unnecessary, because rear->next!=NULL is always "false".
