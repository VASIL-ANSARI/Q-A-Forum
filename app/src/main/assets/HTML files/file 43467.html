Suggestion: SQLiteDatabase has various query methods that return a Cursor. A Cursor can be used to make a CursorAdapter. So one approach might be to wrap the CursorAdapter with a RecyclerView.Adapter and delegate to the CursorAdapter. This would be appropriate especially if you read your data once and the ListView does not have to be notified about data modified by refresh queries.

If you don't know about CursorLoader, look into how that works. That is the best way to get Cursors from queries, since the CursorLoader takes care of running all the database queries in the background. When the CursorLoader completes, take the resulting Cursor and create the CursorAdapter, then take that and create your RecyclerView.Adapter subclass.
