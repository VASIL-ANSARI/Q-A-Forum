There is a substrb function that calculates in bytes (as well as substrc, substr2, and substr4 to get a number of Unicode characters, UCS2, and UCS4 code points). If your database character set is AL32UTF8, remember that characters may require between 1 and 4 bytes of storage-- I don't know off the top of my head what happens if you try to get 10 bytes of data from a string when that would cut off a character in the middle.
