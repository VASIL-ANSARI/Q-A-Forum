Your for loop is always executed exactly five times, no matter how often you encounter a null within the loop's body. You can verify this by manipulating the size of deck to something < 5. If your method were working correct, it should get caught in an endless loop. instead, you will get at least 5 - deck.size many nulls. A quick& dirty fix would be to decremt l in case you encounter a null:

if (deck[index] != null) {
    // ...
} else /* if (deck[index] == null) */ {
    --l;
}

I would recommend a different strategy alltogether. Instead of generating a random index every time you want to draw a card, you could determin the order of all cards up-front, e.g. with the Fisher-Yates-Shuffle. This way, you need only to remember how many cards were drawn and extract the next card from your array/list/queue.
