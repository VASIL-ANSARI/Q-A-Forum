Assuming you're talking about writing Objective C, you specify a superclass of your class in it's @interface declaration after a colon. If you're doing OS X or iOS development, every class you're using probably already inherits from something. For example, your app delegate interface probably looks like:

@interface MyAppDelegate : UIResponder

In this case, MyAppDelegate is a subclass of UIResponder. Or, in the case of a view controller:

@interface MyViewController : UIViewController

Your MyViewController class is a subclass of UIViewController. Even a "do nothing" class that isn't directly integrated into the framework will still inherit from NSObject by convention.
