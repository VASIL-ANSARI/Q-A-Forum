You don't have three categories listed in your example.

I presume there is another column called catC?

Anywho, if you want your records with the highest number of "categories" in the beginning of the result set. You can accomplish that like so 

Order by convert(case when catA=1 then 1 else 0 end, int) + convert(case when catB=1 then 1 else 0 end, int) + convert(case when catC=1 then 1 else 0 end, int) desc

However there is probably a better way to structure your table than catA, catB, and catC for a category based table.
