Borrowing a tip from a module for mongoose that provides an easy way to work with base schemas and differentiate different types of models in the same collection (aka 'tables' in dynamodb) you could add a _type property to each model's schema and use that as a hint to deserialize them with the right class.

For example:

JSON representation for Class B

{
  "foo" : ["some string-a", "some string-b", "some string-c"],
  "bar" : 890,
  "maz" : {"x":-11, "y": 0, "z":89},
  "_type" : "modelA"
}

JSON representation for Class B

{
  "abo" : [{"key": "val", "anotherKey": "anotherValue"}]
  "seq" : [4, 7, 1, 3],
  "_type" : "modelB"
}

You may or may not want to remove _type property when serialising to JSON (if you decide it's useful to have exposed on the object you could just name it type).

I've used this approach in the past in conjunction with a routine to automatically treat all properties beginning with "_" as private fields and just stripping them when serialising to JSON (e.g. _password for password hashes in User objects, and for other metadata that I'd don't intent to directly expose).
