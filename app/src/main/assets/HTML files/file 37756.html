SELECT CONCAT(@pre, SUBSTRING_INDEX(SUBSTRING_INDEX(@x1, @post, t.n), @pre, -1), @post) FROM (SELECT @x1 as haystack) h
CROSS JOIN (
    SELECT 
        1 + a.N + b.N * 10 + c.N * 100 AS n
    FROM

       ,(SELECT 0 AS N UNION ALL SELECT 1 UNION ALL SELECT 2 UNION ALL SELECT 3 UNION ALL SELECT 4 UNION ALL SELECT 5 UNION ALL SELECT 6 UNION ALL SELECT 7 UNION ALL SELECT 8 UNION ALL SELECT 9) b
       ,(SELECT 0 AS N UNION ALL SELECT 1 UNION ALL SELECT 2 UNION ALL SELECT 3 UNION ALL SELECT 4 UNION ALL SELECT 5 UNION ALL SELECT 6 UNION ALL SELECT 7 UNION ALL SELECT 8 UNION ALL SELECT 9) c
    ORDER BY n
) t
WHERE t.n <=
(
    (
        LENGTH(@x1) 
        - 
        LENGTH(REPLACE(@x1, @post, ''))
    ) 
    / LENGTH(@post)
);

Explanation

First step: setting of markers:

We set the separator strings that mark the begin and the end of the part we want:

SET @pre = '@@';
SET @post = ',#';

Second step: getting the needed part

We use an expression similar to yours to get the part we want to have. Because our @pre and @post separators belong to the part we want and SUBSTRING_INDEX eliminates them, we've got to concatenate them afterwards again.

SELECT CONCAT(@pre, SUBSTRING_INDEX(SUBSTRING_INDEX(@x1, @post, t.n), @pre, -1), @post) 
FROM (SELECT @x1 as haystack) h

** Third step: generating numbers**

The trick to fill the part number, here t.n with numbers is to do a cross join with a number generating query:

SELECT 
    1 + a.N + b.N * 10 + c.N * 100 AS n
FROM
    (SELECT 0 AS N UNION ALL SELECT 1 UNION ALL SELECT 2 UNION ALL SELECT 3 UNION ALL SELECT 4 UNION ALL SELECT 5 UNION ALL SELECT 6 UNION ALL SELECT 7 UNION ALL SELECT 8 UNION ALL SELECT 9) a
    ,(SELECT 0 AS N UNION ALL SELECT 1 UNION ALL SELECT 2 UNION ALL SELECT 3 UNION ALL SELECT 4 UNION ALL SELECT 5 UNION ALL SELECT 6 UNION ALL SELECT 7 UNION ALL SELECT 8 UNION ALL SELECT 9) b
    ,(SELECT 0 AS N UNION ALL SELECT 1 UNION ALL SELECT 2 UNION ALL SELECT 3 UNION ALL SELECT 4 UNION ALL SELECT 5 UNION ALL SELECT 6 UNION ALL SELECT 7 UNION ALL SELECT 8 UNION ALL SELECT 9) c
ORDER BY n

This kind of query is easily adaptable to get as many numbers as one wants.
