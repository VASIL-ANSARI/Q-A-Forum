It seems that, the method on which you create the LeveyPopListView triggers more than once when you tap fast to create the list, resulting in creation of more lists than the required.

I think you should insert a check in the list creation method which confirms that 'if the list is already been created than does not create the new one else create one'.

You can do so by declaring the instance of LeveyPopListView as member variable as -

@property (nonatomic, strong) LeveyPopListView *m_lplv;

inside create list

- (void) createLeveyPopList
{
    if(self.m_lplv != nil)
        return;

    self.m_lplv = [[LeveyPopListView alloc] initWithTitle:@"Share Photo  to..." options:_options];
    self.m_lplv.delegate = self;
    [self.m_lplv showInView:self.window animated:YES];
    [self.m_lplv release];
}

Also create a remove method to remove the list from super view as -

- (void) removeLeveyPopList
{
    self.m_lplv.delegate = nil;
    [self.m_lplv removeFromSuperview];
}

Call the above method removeLeveyPopList when you are done with the list, which will remove the list from the superview.

And again when you need to create the list call createLeveyPopList which will check if the list is already created, if YES than it won't create the new else create one.
