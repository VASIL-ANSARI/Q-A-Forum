If you are using python there are good chances you are not aiming at software performance, but you care more about being rapid and agile in development.

Said that, I think the comparison method is quite fair as long as your code is smart enough to avoid bias for one solution.

For instance, a possible improvement for the yield-based version could be removing the range function ad use the xrange function instead. The difference (in python 2.x) is that range builds a list of values (so it must allocate space in memory for it) while xrange builds an iterable object ranging on the given values.
