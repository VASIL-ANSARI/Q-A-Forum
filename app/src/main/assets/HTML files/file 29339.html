In case anyone comes across the same problem, this is what I found:

According to this, Facebook most probably does the following: they submit a (hidden) form with a single parameter with the name "signed_request" and the complete JSON Object encoded as a Base64 string. With the help of the target attribute of the form, the response gets loaded into the iframe automatically. The form can also be created dynamically, and inserted into the DOM with the help of, for example, jQuery.

Another solution would work as follows: Start by sending an AJAX request with JSON data to the external site. The site then constructs the response, caches it and replies with the URL where the response can be loaded. The src attribute of the iframe is then set to this URL, which loads the original response with an HTTP GET request.

I used the second approach, as I have to submit several requests with dynamically changing data, and do not want to edit the DOM each time. I also used the advice from this stackoverflow answer to not leave a browser history entry when loading the response in my iframe.
