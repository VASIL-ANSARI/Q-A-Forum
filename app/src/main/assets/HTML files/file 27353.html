It is useful if there are multiple instances of tour elements inside the page, then when a particular input elements value is changed we need to find out the daily price for the corresponding tour. In that case if you use $('.tour') it will always give the price of the very first tour instead of the one whose input value is changed. In that case we need to find out the tour element which is the ancestor of the element in question that is the reason for using .closest().

If you are sure that there will never be more than one tour element in the page then there is no need to use .closest()
