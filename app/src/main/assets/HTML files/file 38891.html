You most probably want to perform common subexpression elimination. In your example siny is the only expression actually being reused:

>>> expr = pi*sin(pi*x)*sin(pi*y) + sin(pi*y)*sin(pi*z)
>>> print(cse(expr))
([(x0, sin(pi*y))], [pi*x0*sin(pi*x) + x0*sin(pi*z)])

Usually compilers should already do these transformations - at least if you ask it to ignore the non-associativity of e.g. floating point multiplication (by passing e.g. -ffast-math). I don't have any experience with nvcc though.
