I'm not sure what this exception tries to tell. I think that EF can only apply the ?? operator to a mapped object, because everything must be translated into SQL. It has not information of CompanyModels, so it does not know what to do with it.

If so, I think you can solve it by first setting a property, say A, like this:

...
AddressTwo = x.address_2,
A = x.a1.FirstOrDefault() ?? x.a2.FirstOrDefault() ?? 
    x.a3.FirstOrDefault() ?? x.a4.FirstOrDefault(),
...

and giving the OfficeModels class a Company property like this:

public CompanyModels Company
{
    get
    {
        return A != null ? new CompanyModels {Id = A.id, Name = A.name} : null;
    }
}

(I got a similar query working).

As for the second question: EF can not handle enums, so you'll have to work with integers there. Look around here at Stack Overflow for EF + enum work-arounds.
