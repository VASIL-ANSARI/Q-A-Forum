Consider breaking down the problem into simpler steps. For example, the original problem statement that you're given, to evaluate the string

4 5.2 + 5.7 / 6.1 5 - -

is much too hard to do all at once. It is better to decompose the problem into managable pieces. One decomposition is:

  1. Break the string into tokens.

  2. Develop an interpreter that works on the tokens.

This breakdown is the standard approach for solving these kinds of problems. A book like PLAI covers how to do part 2. Part 2 is not difficult: maintain a stack of numbers. As you walk the tokens:

  1. If you see a number, push it onto the stack.

  2. If you see an operator, pop two numbers off the stack, manipulate them with the operator, and push the result back onto the stack.

This process is the heart of an Reverse Polish Notation (RPN) calculator.

The first problem is also not too difficult if you're willing to play with tools like regular expressions. A regular expression can detect patterns in strings and separate strings into pieces. For example, the following example shows how we can split a string into a sequence of words along space boundaries:

var s = "hello this is a test";
var pieces = s.split(" ");

For your purposes, your string won't always be nicely spaced out, so you need to split along numeric and operator boundaries. You need to experiment with regular expression patterns to make the tokenization work the way you want it to.
