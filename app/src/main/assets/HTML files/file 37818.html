Solution: mvn process-classes gwt:run-codeserver

Seemingly there is a bug with gwt:run-codeserver, so if you do not run process-classes before running the CodeServer, the translation files located in:

target/generated-sources/gwt/com.project.shared.client.translation.CommonTranslation.java

will not be discovered and maven fails with a java.lang.NoClassDefFoundError.
