I've experienced this before - not with jQuery but with YUI. I believe the .val() method is prone to returning the value before the change event has propagated - whereas accessing the underlying DOM element's value directly should work. So instead of:

var plantid = $('.drpplants').val();

You'd have something like:

var plantid = $('.drpplants').get(0).value;

You may need to correct that second syntax - I'm not sure that's the best way of accessing the underlying DOM node in jQuery...
