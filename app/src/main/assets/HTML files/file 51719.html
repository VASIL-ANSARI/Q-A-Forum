Like Rajesh says, Javascript code running in your browser is scoped to the window. And since the console runs separately from the window, your Javascript cannot directly access it.

So no, it is not possible to log the text that the user is typing into their Javascript console.

Having said that, if the user calls functions from the console, and you have the opportunity to override those functions, then you could learn a little bit about what the user is doing.

As a proof of concept, add the following Javascript to the page:

var myLog = "";

var realSetTimeout = window.setTimeout;

window.setTimeout = function(a, b){
    myLog += "setTimeout was called from " + (new Error).stack + "\n";
    myLog += "with arguments: " + a + " and " + b + "\n";

    return realSetTimeout.apply(this, arguments);
};

Now if the user was to run setTimeout(function)({ test; }, 1000); from their console, your myLog variable would contain something like this:

Chrome

setTimeout was called from Error
    at window.setTimeout (<anonymous>:3:51)
    at <anonymous>:2:1
    at Object.InjectedScript._evaluateOn (<anonymous>:875:140)
    at Object.InjectedScript._evaluateAndWrap (<anonymous>:808:34)
    at Object.InjectedScript.evaluate (<anonymous>:664:21)
with arguments: function (){ test; } and 1000

Firefox

setTimeout was called from window.setTimeout@debugger eval code:2:51
@debugger eval code:1:1
with arguments: function (){ test; } and 1000

(Of course the code above would also log every call to setTimeout made from the normal page scripts too! So you might want to log only those with a matching stacktrace.)

So it is in fact possible to spy on the user's console commands a little bit, if the user call functions.

Perhaps using getters you could also spy on property accesses.

Whether your users will appreciate you doing this, or whether you will gain any useful information from it, is another matter!
