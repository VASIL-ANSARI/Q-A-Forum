I would actually organize it more "complex".

Every step in the dialog should have a pointer to the next, so you can choose, wich part of the dialog is the next. This makes everything easier, you can even go back in your tree.

So the Dialogpart class holds:
*pointer to the next part
*aviable choices
*func to choose
*func to draw buttons for choices

Implementations like you did are okay for a small application, but horrible for bigger games or actually longer dialoges.
