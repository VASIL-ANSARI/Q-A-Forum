What you have created is a function closure and the variables in that closure will not be garbage collected until after the setTimeout() callback runs.

You can conceptually think of the local variables to a function as individual items that are garbage collected only when no other code that can still be called can reach those variables. So, until your setTimeout() fires, the variable am_i_eaten is still reachable and will not be garbage collected.

This works identically in the browser and in node.js (its literally the same V8 JS engine in Chrome and node.js).
