Most probably you didn't install the right certificate on keychain, or you haven't added the device to the provisioning profile.

Here's a nice link that should help you (has a list of possible issues causing your problem): http://esscomp.co.uk/?tag=cssmerr_tp_not_trusted

Search for cssmerr_tp_not_trusted on google and there's more.

EDIT:

Make sure you installed Apple WWDR certificate (you can download it from Apple Developer Portal, 'Certificates' section *If you do not have the WWDR intermediate certificate installed, click here to download now.). Make sure it doesn't also show up in keychain more than once.

After you install/reinstall WWDR certificate check if your certificate still shows This certificate was signed by an unknown authority. If it doesn't try to build.

If you still have issues you'll most probably need to recreate the certificate, see Certificates/How To section from the provisioning portal for a nice tutorial on how to do this.

You should delete all your invalid certificates to prevent any conflict.
