first_or_create works this way that it will first search the database for the attributes in where clause. if found it will not create new, if fails it will create new one. Now even for new user, the query does not know that the given uid and provider exists or not so it has to run the query first to make sure. see this http://apidock.com/rails/v3.2.1/ActiveRecord/Relation/first_or_create.
