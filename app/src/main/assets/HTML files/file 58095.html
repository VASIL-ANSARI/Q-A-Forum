It is not safe to use pixmaps outside the GUI thread

is the error/warning message you see when the code runs.

In general, no, it's not safe to manipulate a QPixmap on any thread other that that on which the QApplication itself was instantiated. This is because the QPixmap implementation is (for most Os's) tightly bound to the underlying graphics system.

If you have code on a non-gui thread that wants to change/set an icon then use a queued signal or similar mechanism.
