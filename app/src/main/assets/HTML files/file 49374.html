50k words is pretty small, HashMap is likely fine. Only you can say whether it actually is an acceptable solution, since you know your requirements.

Whenever you ask "is this optimal", you need to specify what optimality means for you: minimal space, minimal run time, maximal readability of the code etc. With many of these, optimality isn't actually required: you just need something which is sufficient to meet your requirements, e.g. doesn't use more than X space, doesn't take longer than Y to run.

One minor point is that you can make removal more efficient by not checking hashMap.containsKey first:

if (hashMap.remove(word) != null) {
    Toast.makeText(MainActivity.this, "ok", Toast.LENGTH_SHORT)
            .show();
} else {
    Toast.makeText(MainActivity.this, "not found", Toast.LENGTH_SHORT)
            .show();
}

which simply avoids looking up the value in the map twice. Note that this only works because you are inserting non-null values into the map when you construct it.
