There are (or were) two good reasons for making window.open() return a fake window object when popup blocker is enabled:

  1. Some site / apps might try to detect popup blocking and force the user to disable the blocker and endure ads and other abuse

  2. Back when we implemented popup blocking, we often saw scripts that would break if there was no return object. I.e. if they wanted to check win.closed but 'win' was set to undefined.

This "feature" may no longer be required since some browsers implement popup blocking differently.
