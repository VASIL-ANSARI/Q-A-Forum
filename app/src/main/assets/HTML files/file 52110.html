I'll focus on Table2 being nil.

It's nil because you're not creating it.

Go into your terminal and type rails g controller table2 create

open that file and inside of the empty create method you will create the new Table2 object.

You can do this many ways, but the most important thing to remember when you have associations is that you need to define the association when you're creating it.

I'll give you an example:

Let's say I have a User model

User Model

class User < ActiveRecord::Base
has_many :posts

end

Now let's say I have a Post model and I want the user to be able to create many posts.

Post Model

class Post < ActiveRecord::Base
belongs_to :user

end

Now this is the important part where we will utilize your table2 controller

When I create a new post, it has to belongs_to a user, therefor we will do this

Post Controller

class PostsController < ApplicationController

  def create
    #store the object that has many posts as a variable
    @user = current_user

    #the important part, when you create a post you need to do this
    @post = @user.posts.create(post_params)
  end

end

Whats happening here is, Rails is creating the new post UNDER ownership of the @user. That is how you establish a relationship by association.
