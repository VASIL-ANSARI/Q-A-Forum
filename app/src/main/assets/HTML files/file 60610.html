something like:

public class Test {
    public static void main(String[] args) {
        System.out.println(Number.values()[Number.One.getNumber() + Number.Two.getNumber()]);
    }
}

public enum Number {
    Zero(0), One(1), Two(2), Three(3);
    private final int someIndexThatLooksLikeACardinalButIsNot;

    private Number(int card) {
        this.someIndexThatLooksLikeACardinalButIsNot = card;
    }

    public int getNumber() {
        return this.someIndexThatLooksLikeACardinalButIsNot;
    }
}

Note before down vote:

If your Enum would be defined in the exactly order, then there is no need for an int variable for cardinality value, but IT IS a good practice not relying to the order of the defined fields in the enum...(It has to work even if the order changes.)
