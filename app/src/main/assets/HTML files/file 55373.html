It does not work because the bundle extension's load method responsibility to load service configuration related to the bundle it belongs to and only for that. The container you get as the argument of the load method is just a copy of the app container, it does not contains everything (only just parameters) [#using-the-load-method].

Using the load() Method

In the load() method, all services and parameters related to this extension will be loaded. This method doesn't get the actual container instance, but a copy. This container only has the parameters from the actual container. After loading the services and parameters, the copy will be merged into the actual container, to ensure all services and parameters are also added to the actual container.

As you said you can use the prepend method to provide a sensible default configuration for other bundles and ease the use of yours [prepend_extension].
