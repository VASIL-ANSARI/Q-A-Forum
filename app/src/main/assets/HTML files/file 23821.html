It got moved to a gem: https://github.com/rails/activerecord-session_store

See Extraction of features to gems in the edge guide.

The reason it was removed is mentioned in this post:

it isn't scalable. ... Once your application receives a large amount of traffic, the sessions database table is continuously bombarded with read/write operations.

So, if scalibility and performance are important, you might want to look at other ways to manage that data.
