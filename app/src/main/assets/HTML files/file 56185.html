Suggestions:

  1. Never draw directly within a JApplet or other top-level window.
  2. Instead draw in the paintComponent of a JPanel that is displayed within the applet. The Swing tutorials will show you how.
  3. Be sure to call the super.paintComponent(g) method within your override, and again read the Swing tutorials to see why. For more tutorials see: Swing Info
  4. This is Swing -- use a Swing Timer to drive your animation, not threads.
  5. If you ever do use Threads, never call Thread#stop() or use any other deprecated methods. Please read Why is Thread.stop deprecated?.
  6. Please look at this answer for an example of Swing animation using a Swing Timer.
  7. Unless this is for a class assignment, don't create JApplets as this is a dead technology, something even Oracle will tell you.
  8. To display text in the background use the java.awt.Graphics method for writing text: drawString(...). Either that or place a JLabel over your background image.
