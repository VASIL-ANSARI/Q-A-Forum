And create the relevant view file, edit.html.erb:

<%= form_for (@profile, method: ) do |f| %>

As for the issue itself, I would suggest maybe looking at the log files. On your terminal run tail -f /path/to/app/log/* and then try and submit the form. It would be unusual that you don't see anything here, if that is the case, use your browser's developer tools to inspect the form. You should have something like:

<form id="profile-form" action="/profiles/1" method="post">
...
</form>

Look out for action="/profiles/1 where 1 is the ID of the Profile entity.
