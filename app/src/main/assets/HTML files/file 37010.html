A mutex synchronizes access by virtue of only being held by a single thread at any given time. Your code will prevent multiple instances of FileWriter.CreatePDFFile() from executing at the same time.

Because it uses a named mutex it will also synchronize access across processes on the system, e.g. all instances of all applications that use the FileWriter class.

Your description of the simulation is confusing. The mutex will have no effect if you are only running one process that uses the class, and that process has only a single thread.
