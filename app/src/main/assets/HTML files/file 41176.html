You cache invalidating mechanism is not proper as it can return same value in multiple refreshes. You are using the md5 of a random num between 1 & 50 so multiple attempts can return the same random value.

$(document).ready(function () {
    $.getJSON('json/summary.json?nocache=' + new Date().getTime(), function (data) {

        var sold = data.sold;
        var price = data.averagePrice.toFixed(2);

    });
});

Another option is to set the cache option globally for all jQuery ajax requests like

$.ajaxSetup({cache:false})

then

$(document).ready(function () {
    $.getJSON('json/summary.json', function (data) {

        var sold = data.sold;
        var price = data.averagePrice.toFixed(2);

    });
});

But if you don't want to cache the particular resource value, then you can disable the cache from the server side by setting cache headers like no-cache and no-store
