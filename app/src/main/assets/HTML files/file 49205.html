You can use nested serialization.

DRF offers a feature called nested serializers.

For your example you could do the following:

class PendingRequestSerializer(serializers.ModelSerializer):

    driver = serializers.SlugRelatedField(
        slug_field='username',
        queryset=User.objects.all()
    )

    passenger = serializers.SlugRelatedField(
        slug_field='username',
        queryset=User.objects.all()
    )

    ride = RideSerializer()

    class Meta:
        model = PendingRequest
        fields = ('passenger', 'driver', 'ride', )
        read_only_fields = ('passenger', )

This will produce the normal JSON and instead of showing the PK of the ride instance in the ride field, it will nest the entire ride object. You will of course need to write that RideSerializer for that to work.
