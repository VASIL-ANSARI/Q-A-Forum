Yep.

Will i be able to write my own low level functions to access a drive whose file format is not known to android Storage Access FrameWork?

You'll have to write your own USB drivers using the low level USB APIs (as you said, Android doesn't know how to read your USB drive)

Lets say i create my own drive format for security reasons, how will Android present the dialog box showing the user the files contents so they can pick a file granting the app permission to access that file it it doesnt know the file format of the drive?

Nothing in the DocumentsProvider API controls how your files are read from disk (if they even are read from disk - in many cases, like with the Google Drive DocumentsProvider, they are read from a remote server). As long as fulfill the API by returning a Cursor for the root from queryRoots(), a Cursor with the proper fields from queryDocument() for a single document and a Cursor with a list of documents from queryChildDocuments(), then the Storage Access Framework will list your files correctly.

Just make sure you provide a valid ParcelFileDescriptor to openDocument() which would allow read/write access to your custom file format system.
