From description it is not clear, what is the goal of listening the CellContentClick event, and on what manipulations you want to track the selected column index.

May be the CurrentCellChanged event will be more suitable in your case?

And i recommend you to use the switch statement for your purpose, to avoid numerous if blocks, like:

private void dataGridView1_CurrentCellChanged(object sender, EventArgs e)
{
    var gridView = sender as DataGridView;
    if (gridView == null || gridView.CurrentCell == null) { return; }

    switch (gridView.CurrentCell.ColumnIndex)
    {
        case 6:
            // edit
            break;
        case 7:
            // delete code
            break;

    }

}
