In general, it's not a good idea to use the same JVM in latency-optimized workload (your REST API) as well as throughput-optimized workload (your CPU-intensive work). It will be difficult to tune the garbage collection in a way that's specific to the type of your workload. Simpler and smaller JVMs will be more predictable.

Specifically to your question, make sure you pass the -XX:+UseThreadPriorities flag as described here, and run the application as root if on Linux.
