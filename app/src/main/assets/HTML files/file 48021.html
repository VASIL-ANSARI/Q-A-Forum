You don't need to set the paths separately for clang code completion to work. You just need to re-build (i.e. clean follow by build) your project once.

If everything is done properly, you should see in CodeLite's build log lines similar to:

codelite-cc g++ ... (or codelite-cc clang++...) The codelite-cc utility will collect the search paths (and macros and other flags) passed to the compiler and will pass them back to CodeLite, so it will pass it to libClang for getting the proper code completion for your code.
