I have ran into the same issue.

It appears when Knockout bindings which remove video elements also break them.

You probably already fixed it, but for anyone else, this is fixed by making sure 'with' and 'if' bindings don't apply to a video.

With your code this would mean something like:

<div id="assessmentIntroPanel" data-bind="with: currentStep">
  <div data-bind="video: videos, playerId: 'videoPlayer'">

  </div>
</div>
<video id="videoPlayer" class="video-js vjs-default-skin" width="430" height="267">
</video>

So your UI will be more complicated... but now as the video element is never removed it should work.
