We can't know why the contributors to jQuery didn't do something. We can guess that they didn't want to make a method for every property and attribute (to avoid bloating the size) so they chose a generic way of addressing all of them.

You can roll your own methods easily enough if it matters to you:

$.fn.disable = function() {
    return this.prop("disabled", true);
}

$.fn.enable = function() {
    return this.prop("disabled", false);
}

Or, a single method:

$.fn.enable = function(val) {
    if (val === undefined) {
        val = true;
    }
    return this.prop("disabled", !val);
}
