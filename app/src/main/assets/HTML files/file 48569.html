Be sure you're plugin key is correct. Near the top of your atlassian-plugin.xml you should find a line that looks like: <atlassian-plugin key="${project.groupId}.${project.artifactId}" ...>.

The project.groupId is the groupId in your pom.xml, and the project.artifactId is the artifactId in your pom.xml.

So lets say your groupId was com.atlassian.plugins and your artifactId was tabpanel, then using your web resource key which is tabpanel-resources (as shown in this line <web-resource key="tabpanel-resources" name="tabpanel Web Resources">), the resulting requireResource statement should be:

#requireResource("com.atlassian.plugins.tabpanel:tabpanel-resources")

This should be put at the top of your Velocity template and the macro will put the resources into the head element of the resulting HTML.

Possible other cause

The #requireResource part is a velocimacro which is made accessible by the webResourceManager object being available in the Velocity template. So if it doesn't work, make sure an instance of JiraWebResourceManager is put into a context map for the velocity template with the key of "webResourceManager".
