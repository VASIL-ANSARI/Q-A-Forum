Instead of putting your development files in ~user/Sites, I would put them in the webserver root, which is in /Library/WebServer/Documents. I've had better luck with that.

Then, make sure that your AllowOverride All in /etc/apache2/httpd.conf is inside the directory tag marked <Directory "/Library/WebServer/Documents">. There are several AllowOverride directives in httpd.conf, the one you want to modify is around line 217ish if you use my approach.

Also make sure that the line LoadModule rewrite_module libexec/apache2/mod_rewrite.so is uncommented.

Finally make sure you restart apache with sudo apachectl graceful.

If you do want to use your user sites directory, this blog entry has some tips. http://johnattebury.com/blog/2012/07/upgrading-to-mountain-lion-forbidden-403-you-dont-have-permission-to-access-username-on-this-server/
