Please take a look at the ChangeVersion example I made:

public void manipulatePdf(String src, String dest) throws IOException, DocumentException {
    PdfReader reader = new PdfReader(src);
    PdfStamper stamper = new PdfStamper(reader, new FileOutputStream(dest), '4');
    stamper.close();
    reader.close();
}

In this example, I create a PdfReader and a PdfStamper. In the PdfStamper, I tell iText that I want the version of the file to change to version 4.

Once I close the stamper, the magic is done. Please check the file other_version.pdf to convince yourself that this works.

One observation: your original file OCR.pdf has a compressed cross-reference table. If you would open this file in a text editor, and manually change the version number from 1.6 to 1.4, Adobe Reader will tell you that the file is an 1.5 file because compressed cross-reference tables didn't exist in PDF 1.4. They were introduced in PDF 1.5.

In this case, iText will uncompress the cross-reference table, resulting in a valid 1.4 PDF file. However: iText doesn't downgrade all new functionality automatically. If a specific PDF contains newer functionality (e.g. OCGs), then iText won't remove those OCGs if you change the PDF version to a version predating the introduction of OCG.
