The property tree is a hierarchical data structure where each node can have an ordered sequence of sub-nodes. In this sense the property tree is more similar to XML than flat ini files. And while it can be initialised from both ini files and XML it represents the internal data all the same, and a query of "x.y.z" has a special meaning of selecting the z child of the y child of x.

It boils down to the string_path class of the property tree which has the default separator of .,

/// Default path class. A path is a sequence of values. Groups of values
/// are separated by the separator value, which defaults to '.' cast to
/// the sequence's value type. The group of values is then passed to the
/// translator to get a key.

Overall, either avoid dots in the names of your items when you use an ini file, or change the separator by constructing paths explicitly:

configTree.get<std::string>(ptree::path("B.x", '/')); // using non-default separator
