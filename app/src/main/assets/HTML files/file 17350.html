You could use some sort of encryption. Then obfuscate the encryption keys by some calculation so that they are not easily retrievable by reverse engineering efforts. Make sure the keys and decrypted data is disposed immediately after usage.

If you retrieve the passwords through network, using authentication and SSL and obfuscated URLS in the source code, then you can detect attempts to retrieve the passwords and ban the connections from that source. But this is much added effort and introduces potential problem if your server is down.
