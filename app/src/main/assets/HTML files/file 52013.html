Found the solution here. The problem can be solved by changing Name to x:Name like this:

<VisualState x:Name="MagicString">
    #Storyboard etc.
</VisualState>

will create a field in the view class which can be acessed by codebehind. So the call works like this:

VisualStateManager.GoToElementState(element, MagicString.Name, true);
