You probably need to enforce data integrity not only on INSERT into Table B, but also on UPDATE and DELETE in both tables.

Anyway options are:

  1. FOREIGN KEY CONSTRAINT on Table B
  2. TRIGGERs on both tables
  3. As a last resort if for some reason 1 and 2 is not an option STORED PROCEDUREs for all insert, delete update operations for both tables

The preferred way to go in most cases is FOREIGN KEY CONSTRAINT.
