Here is some feedback:

  1. It's not that normal to have a lot of whitespace after property names. Maybe it's a copy/paste problem.
  2. Objects as shown by the {} are not arrays, FYI. SO you might not wants to call them arrays. You can make an array literal by using [], if that's what you want.
  3. document.getElementById( 'someID' ); should NOT have a semi-colon at the end. That's your biggest mistake.
  4. You sometimes use comma-before syntax and sometimes use comma-after. You could probably just stick with one. Personally, most of my JS is in node and comma-before is pretty normal there. Either works though. http://css-tricks.com/7741-commas-before/
  5. I put this in my code example, but didn't mention it up here. Braces should always be on the line with the parent. This is because of automatic semi-colon insertion, which can screw you up if you put a return on the line before {}. See: http://robertnyman.com/2008/10/16/beware-of-javascript-semicolon-insertion/. Thanks Michael for mentioning it below!
