The second method is undefined behaviour due to violating the strict aliasing rule. Even though struct one and struct two have the same layout, it is not permitted to use an lvalue of type struct one to access an object of type struct two or vice versa.

In general, accessing an object via an expression of a different type is only allowed for using a character type to access another object; or via a union.

An alternative would be:

memcpy(&s1, &s2, sizeof s1);
