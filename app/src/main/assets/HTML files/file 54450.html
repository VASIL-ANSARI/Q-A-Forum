The stack trace clearly states that there is a class in the tree that is not serializable: com.cogentautomation.view.ViewMain$23. You have to read this path like this: com.cogentautomation.view.ViewMain is the outer class, and the $23 denotes the 23rd anonymous inner class. Since there are a lot anonymous classes in ViewMain, you may have a difficult time figuring out which class it is.

Easiest thing to do, would be to install the JDK (get the sources of "java") and set an exception breakpoint at java.io.ObjectOutputStream.writeObject0(Unknown Source).

On a side note, this kind of GUI will make your life difficult in the future. As you can see, debugging is more difficult of the massive amount of anonymous classes in one file and the file length must be excruciating long. I suggest that you apply a few techniques to slim down ViewMain like the Command-Pattern or Chain of Command. There are more, maybe you spend some time on Wikipedia to get to know them.
