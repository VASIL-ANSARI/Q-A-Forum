You don't really need a POJO for this. Just create your view model with the bindable property you need. ObservableInt does everything for you and is handled as if it was an int.

public class Pressure {
    public final ObservableInt value = new ObservableInt();
}

In your layout you then only need to set binding:textValue="@={pressure.value}". Using android:text would be possible too, but not too good, because setting an int to it should stay the default behavior.

@BindingAdapter("textValue")
public static void bindIntegerInText(TextView text, int value) {
    text.setText(String.valueOf(value));
    // Set the cursor to the end of the text
    text.setSelection(text.getText().length());
}

If you want to use the validation for the text color, you'd have to use an observable expression there as well. In you case it gets evaluated exactly once at the beginning, because the databinding doesn't know anything about the value underlying your validation.

You could add an ObservableBoolean isValid to Pressure and update its value anytime the value changes. For this you can register your own Observable.OnPropertyChangedCallback to value and set isValid in there.
