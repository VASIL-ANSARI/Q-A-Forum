Your query is almost OK, only TABLE keyword is redundant.

UPDATE
  sl_adr a, sl_address b
SET
  a.fk_adrID = b.id
WHERE
  a.street1 = b.street1
  AND a.postal_code = b.postal_code
  AND a.city = b.city
  AND a.fk_countryID = b.fk_countryID

also, this one is possible -

UPDATE sl_adr a
JOIN sl_address b
  ON a.street1 = b.street1
    AND a.postal_code = b.postal_code
    AND a.city = b.city
    AND a.fk_countryID = b.fk_countryID
SET
  a.fk_adrID = b.id
