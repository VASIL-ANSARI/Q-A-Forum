A few suggestions:

  1. Element IDs must be unique. You shouldn't use "t" for each input ID. You could change it to id="t_$i_$x" or something like that to make it unique.

  2. You need to let your server side script know that t is an array. To get all of the values from the form submitted, change the name=t in the<input> element templates to name="t[]".

  3. Step 2 will break your validation function. I would change onchange=ty() to onchange="ty(this)". You will then have to change your ty() function implementation to:

    function ty(el) {
    if(!/^-?\d*$/.test(el.value)) {
        alert("Number of heads per model value must be numeric!");
    } 
}

    This consequently simplifies your function by removing the need to get the element again (you simply pass an instance of it in when you call it in the onchange event).

  4. This isn't necessarily required, but it's a good practice to always enclose your HTML tag attribute values in quotations, as I've shown in my suggestions above.
