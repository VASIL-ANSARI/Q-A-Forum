You can do this in various ways:-

Make reusable functions as service

app.service('GetQCQAData' ,function() {
  //your code here
});

Then in your external function

function QCQAIssues() {
     var $injector = angular.injector();
     var GetQCQADataFn = $injector.get('GetQCQAData');
 //your code here
}

**Prototype Way ** This is not preferred because you cannot gurantee if the controller was executed before QAQAIssues() call (call vs define - definition is ok)

Your external function

function QCQAIssues() {
  this.GetQCQADate();
  //this will be enhanced later 
  //you must be absolutely sure that QCQAIssues 
  //are called after the controller does its thing
}

In your controller

app.controller('QCQA', function ($scope, $http) {
   var GetQCQAData = function () {
     ///your code
   };

    $scope.GetQCQAData = GetQCQAData;

    QCQAIssues.prototype.method = GetQCQAData;// 
}

In general - I would advise you against mixing different namespaces and advise you to find a way to encapsulate your external function inside AngularJS. Think that QAQCIssues is on the top level window object.
