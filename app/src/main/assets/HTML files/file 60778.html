If you do not want to use an extension method and change all the already implemented code that calls the string indexer then you could roll your own provider and do it there. See Implementing a Session-State Store Provider. You can then implement your "guid prefix" there.

Here is an example of an extension method. I have no idea where you are getting guid from, you never mentioned this but you can take this example and add it yourself.

public static class HttpSessionStateExtension
{
    public static object Get(this System.Web.SessionState.HttpSessionState session, string key)
    {
        var guid = ???; // no idea where or how you are getting guid, you never explained that in your question
        return session[guid + key];
    }
}

// in other methods you can then call it the same as you would the indexer
var result = System.Web.HttpContext.Current.Session.Get("foo");
