I tend to do this with templating. I create a simple templating engine that does basic string replacements like this

    function renderTemplate(str, obj) {
        var i, r, val, outStr=str;

        for(i in obj) {
            if (obj.hasOwnProperty(i)) {
                r = new RegExp("{" + i + "}", "g");
                val = obj[i] || "&nbsp;";
                val = val.toString().replace(/^null$/, "&nbsp;");
                outStr = outStr.replace(r, val);
            }
        }

        return outStr;
    }

Then I'll create a template string like this

var template = "Hello <span class=\"name\">{name}</span> you are {age} years old";

Given an object that contains name and age, you can render the template to html like this

var html=renderTemplate(template, {name: "Alice", age: 38});

The keys in the object passed in in the second argument are searched in the template wrapped in curly braces ("{" and "}")

Lastly, I create a new element and set this string to the innerHTML and append it to my document

var newLI = document.createElement("li");
newLI.innerHTML = html;
ulNode.appendChild(newLI);

this renderTemplate is an early creation of mine and I have new versions burried in code that are more fault tolerant, but this is an ok example.

Note: There are other templating solutions in the wild. This is my home grown one. As a comment on your OP said, Mustache is one such.
