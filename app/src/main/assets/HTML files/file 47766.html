You can have multiple partition strategies:

  • Round-robin ('id%n==0')
  • Time-based
  • Custom (using a mapping table)

Furthermore, you will need to expand on this subject. Regarding the above-mentioned strategies I'll try to quickly and briefly mention some details.

  1. Round-robin : Considering that you have n partitions any new item created having and id of m should go in partition with id m%n. For example having 4 partitions (identified in a 0-based index) called p0, p1, p2 and p3, and having had created item with id 573 this should go in partition p1 because the remainder of dividing 573 to 4 (4 being the number of partitions) will yield 1.

  2. Time based : Used sometimes in statistics data, data-warehousing, big-data etc. You will have partitions for several segments of time, for example a table for each month, or day, or hour. Newly created items will go in their corresponding partition based on the creation time.

  3. Custom : You will define your own partition strategy and use a mapping table to map the item id and the partition id in which it resides. This is the most flexible but might not be easiest to implement or the most performant.
