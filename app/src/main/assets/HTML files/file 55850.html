I think the problem is these lines:

DbBroker db = new DbBroker(this, "my_database", null, 1);
SQLiteDatabase sqldb = db.getWritableDatabase();
public int test = db.insertTest("New test", "Test created from app");

Variables will be sorted in compiler to optimise the allocation memory. so the order of these can be changed while complie. So the scenario can be the line #2 is called first and db instance now is null so the exception was thrown.

To fix it, place it in specified method, such as onCreate(), ... so these command can be execute sequently.

Note: you are missing a close bracket at the end of sqlite command.

create table tests (id INTEGER PRIMARY KEY, publicid INTEGER, testname TEXT, description TEXT);
