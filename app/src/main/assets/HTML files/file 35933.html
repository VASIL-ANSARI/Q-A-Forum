I was struggling with what I think was the same problem as you:

  • My WebAPI method was returning a valid SAS URL to FineUploader;
  • FineUploader would attempt to upload the file to my local server instead of to the SAS URL.
  • Curiously, the SAS URL was showing up somewhere in the URL that FineUpload was ultimately using.

The problem turned out to be that WebAPI was automatically serializing the SAS URL as JSON. This resulted in the SAS URL being malformed -- a few extra characters were being added -- and FineUploader was confused by the resonse. In your case, it looks like the SAS URL is being serialized as XML, based on the schemas.microsoft.com text that's present.

With that understanding I bypassed WebAPI's automatic serialization. Below is a working implementation of my GET /api/sas method. Note: GetSasForBlob is pulled directly from the Sample Azure Server.

public HttpResponseMessage Get(string bloburi, string _method)
{
   var accountAndKey = new StorageCredentials(STORAGE_ACCOUNT_NAME, STORAGE_ACCOUNT_KEY);
   var sas = GetSasForBlob(accountAndKey, bloburi, _method);

   var content = new StringContent(sas);
   return new HttpResponseMessage {Content = content};
}
