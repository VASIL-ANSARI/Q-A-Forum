Depends on what you mean by "sort by column". If you want to treat each column independently and sort each one, the following code will do that:

    public static double[][] sortColumns(double[][] m) {
        int mrows = m.length;
        int mcols = m[0].length;
        double[][] retVal = new double[mrows][mcols];

        // process array column by column
        for (int i = 0; i < mcols; i++) {
            // copy column i from m into new array
            double[] mcol = new double[mrows];
            for (int j = 0; j < mrows; j++) 
               mcol[j] = m[j][i];

            // sort array
            Arrays.sort(mcol);

            // write sorted column to return value
            for (int j = 0; j < mrows; j++)
                retVal[j][i] = mcol[j];
        }

        return retVal;
    }

So if the input is:

1.0 2.0 3.0 
9.0 8.0 7.0 
5.0 8.0 3.0

The output will be:

1.0 2.0 3.0 
5.0 8.0 3.0 
9.0 8.0 7.0

If instead you want to sort the rows of the matrix by a particular column value, then the following code will do that:

    public static double[][] sortByColumns(double[][] m, int c) {
        int mrows = m.length;
        int mcols = m[0].length;
        double[][] retVal = new double[mrows][mcols];

        // copy into return value
        for (int i = 0; i < mrows; i++)
            retVal[i] = Arrays.copyOf(m[i], mcols);

        Arrays.sort(retVal, new Comparator<double[]>() {
            public int compare(double[] a, double[] b) {
                return Double.compare(a[c], b[c]);
            }
        });

        return retVal;
    }

So if the input is:

1.0 3.0 2.0 
7.0 4.0 9.0 
3.0 2.0 1.0

Then if we sort by column 1, then the output will be:

3.0 2.0 1.0 
1.0 3.0 2.0 
7.0 4.0 9.0

Each of these solutions make a copy of the original array, and therefore "keep the original array in tact".
