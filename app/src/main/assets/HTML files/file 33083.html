No, they are not the same:

glOrtho(1, 1, 1, 1, 1, -1);

will just generate a GL_INVALID_OPERATION error, and the commend has no further effect. The error occurs when left==right or top==bottom or near==far. (If it did not check for that condition, a division by zero would occur).

glOrtho(-1, 1, -1, 1, 1, -1);

is effectively doing nothing, as it will multiply the current matrix by an identity matrix.

It might appear as if those commands have the same result, but what actually happens is quite different.
