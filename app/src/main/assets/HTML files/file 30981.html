If you suspect a problem at your sendSynchronousRequest, you should consider at least three things:

  • Look at the NSHTTPURLResponse object. What did it report? Notably, for HTTP requests, you'll see a statusCode, which can be informative.

  • You should pass it an NSError address pointer, too, and make sure that the error was nil, too:

    NSError *error = nil;
NSHTTPURLResponse *urlResponse = nil;
NSData *responseData = [NSURLConnection sendSynchronousRequest:request returningResponse:&urlResponse error:&error];

if (!responseData) 
    NSLog(@"sendSynchronousRequest error: %@", error);
else {
    NSLog(@"sendSynchronousRequest response = %@", urlResponse);
    NSString *rawData = [[NSString alloc] initWithData:responseData encoding:NSUTF8StringEncoding];
    NSLog(@"responseData = %@", rawData);
}

  • What did you get as your rawData string?

By the way, if you're wondering, you want the Content-Type as the header field, not as the value.

// [request addValue:@"Content-Type" forHTTPHeaderField:@"audio/x-flac; rate=16000"];
[request addValue:@"audio/x-flac; rate=16000" forHTTPHeaderField:@"Content-Type"];
