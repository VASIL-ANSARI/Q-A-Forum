In general, FindControl is rarely useful unless you only want to go one layer deep.

EDIT:

From the link provided:

public static T FindControl<T>(System.Web.UI.ControlCollection Controls, string ControlName) where T : class
{
     T found = default(T);

     if (Controls != null && Controls.Count > 0)
     {
          for (int i = 0; i < Controls.Count; i++)
          {
               if (Controls[i] is T && Controls[i].ID == ControlName)
               {
                    found = Controls[i] as T;
                    break;
               }
               else
                    found = FindControl<T>(Controls[i].Controls);
          }
     }

     return found;
}

And in your case, you would use (in C#):

var uName = FindControl<TextBox>(LoginView1.Controls, "UserName");

The generics usage in VB.NET is not something I'm very familiar with. You could remove the generics portion of this and just use Control.
