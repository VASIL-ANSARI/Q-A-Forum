Try using Port Forwarding.

In Ubuntu (details here), execute these 3 commands:

sysctl net.ipv4.ip_forward=1

    iptables -t nat -A PREROUTING -p tcp -d yourIP --dport   
portNumberOfTheServerRunningONTheEmulator -j DNAT --to-destination yourEmulatorsIP

iptables -t nat -A POSTROUTING -j MASQUERADE

I tried this on my machine my emulators ip was 192.168.56.101 and my PC's IP address was 10.0.20.159. So I run these commands: 

 sudo isysctl net.ipv4.ip_forward=1

assume the server program is running on port 5555

sudo iptables -t nat -A PREROUTING -p tcp -d 10.0.20.159 --dport 5555 -j DNAT --to-destination 192.168.56.101

iptables -t nat -A POSTROUTING -j MASQUERADE

So then I tried to connect to the emulator from another machine on the network typing this address: 10.0.20.159:5555 (the result was connection was reset because I dont have actual server program running on the emulator) but when I try with d/t port say 10.0.20.159:5595 the result is unable to connect so my guess here is the PC was able to reach the emulator through the port forwarding)
