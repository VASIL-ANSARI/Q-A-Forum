The class StandardCharsets gives you a list of encodings / charsets that are "guaranteed to be available on every implementation of the Java platform."

This list doesn't contain the Windows encodings but for most common Java versions on Windows, Mac and Linux, Cp1251 is available.

Note that you'll get a UnsupportedCharsetException or UnsupportedEncodingException when it's not available, so the code above is safe (in the sense that it won't produce garbage).

If you want to be really safe, the common approach is to use only UTF-8 encoded data in your projects.
