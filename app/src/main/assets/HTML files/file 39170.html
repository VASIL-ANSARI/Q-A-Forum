libvlc/video/set_format_callbacks is meant completely for different things thatn libvlc/media_player/set_hwnd. While the use of the latter is fairly obvious (selecting target screen area), the former (according to this source) is callback replacement for libvlc_video_set_format, well, I didn't check the libvlc docs though, but it would probably be described there. I'm no expert in libvlc, but, IMHO, they are far from each other.

Regarding other functions that were visible in comments in the code:

  • libvlc_media_player_set_nsobject -- uses NSView (Cocoa)
  • libvlc_media_player_get_nsobject -- returns NSView (Cocoa) previously set by the former
  • libvlc_media_player_set_agl -- likewise, but for AGL (Apple OpenGL)
  • libvlc_media_player_get_agl -- likewise, but for AGL (Apple OpenGL)
  • libvlc_media_player_set_xwindow -- likewise, but for XWindows (genric Linux+XWin)
  • libvlc_media_player_get_xwindow -- likewise, but for XWindows (genric Linux+XWin)
  • libvlc_media_player_set_hwnd -- likewise, but for Microsoft Windows
  • libvlc_media_player_get_hwnd -- likewise, but for Microsoft Windows

(see the docs about them)

So, it depends on what platform you are trying to use the libVlc, because that dictates the way the VLC can draw upon the screen.

If you are building for whatever on MS Windows - use HWND pair.
If you are building for Linux with XWindows - use XWindow pair.
If you are building for Cocoa on MacOS/iOS - use NSObject/NSView pair.
If you are building for OpenGL on MaxOS/iOS - use AGL pair.

On MS Windows, you must use HWND. Even if you are using WPF/GTK#/whatever. Every windows in MS Windows uses HWND in its roots. Dig it out from your UI library. It is somewhere there. But it might be unsatisfactory. For example, WPF (unless you do some tricks) uses exactly one HWND handle for each Window object. Passing this HWND to VLC would result in VLC drawing all other the window, not just the region you would want. That must be solved by different means - I don't know how yet.
