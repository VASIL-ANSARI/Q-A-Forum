After some additional research, using System.Security.Cryptography is a very poor choice because Windows already has a builtin mechanism involving RSA encryption called Code Signing. Signing your code allows users to know where code comes from and if it has been tampered with.

ShellExecute is a much better choice than CreateProcessAsUser because it can handle ERROR_ELEVATION_REQUIRED exceptions by using UAC. Most users and developers hated (and probably still do) when Vista came out with this but I highly encourage you to watch UAC - What. How. Why. where both the Project Lead and System Architect of UAC explain why it was needed and how it has increased security.
