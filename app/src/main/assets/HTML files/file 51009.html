Use the Repository Pattern

A Repository mediates between the domain and data mapping layers, acting like an in-memory domain object collection.

  • First, make sure you have a database-independent model. Whether or not you should build a fully-fledged domain model or just use a simple property bag model is another question, but irrelevant here.
  • Build a repository that accepts your model objects for create and update operations, and returns model objects from read operations. The DAOs are now an internal concept of the repository and not visible to the rest of the application.
  • Now the repository is responsible for mapping DAOs to model objects and vice-versa. If you can implement the mapping in a generic way, extract it from the repository into a dedicated mapper and use the mapper from the repository to adhere to the Single Responsibility Principle.
