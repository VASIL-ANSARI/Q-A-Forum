The proposed solution is correct because you do not have an unlimited number of coins of each denomination:

[I]t is often helpful to bring cash.

The following n lines each contain one integer, the value in cents of each bill or coin that you have.

That is why the loop in the proposed solution decreases:

for (int v = 30001 - C - 1; v >= 0; v--)

This avoids using the same coin twice because when it looks at dp[v] to construct dp[v+C], dp[v] has been constructed without using the current coin C.
