OK, Solved!

The problem was (apparently) related to the class path. I'm new to Eclipse development, so it's getting clearer and clearer for me each day how it works.

To solve similar problems, do the following:

  1. Add the dependency to your gradle build compile ("com.fasterxml.jackson.datatype:jackson-datatype-jsr310:2.5.1")

  2. You have to refresh your class path. With gradle, to do that, run gradle cleanEclipse && gradle eclipse

  3. Refresh your eclipse. On mine, I really had to restart it.

After this, it did the magic: the LocalDateTimeDeserializer was there for me to import.
