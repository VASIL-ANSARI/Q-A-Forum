According to the comments in my answer, I also add this way:

if ("ontouchstart" in document.documentElement) {
    //do stuff
}

A mobile device is touch-sensitive so if the touch event exists it has to be mobile.

Javascript

You can detect mobile devices this way with Javascript:

var isMobile = {
    Android: function() {
        return navigator.userAgent.match(/Android/i);
    },
    BlackBerry: function() {
        return navigator.userAgent.match(/BlackBerry/i);
    },
    iOS: function() {
        return navigator.userAgent.match(/iPhone|iPad|iPod/i);
    },
    Opera: function() {
        return navigator.userAgent.match(/Opera Mini/i);
    },
    Windows: function() {
        return navigator.userAgent.match(/IEMobile/i);
    },
    any: function() {
        return (isMobile.Android() || isMobile.BlackBerry() || isMobile.iOS() || isMobile.Opera() || isMobile.Windows());
    }
};

Now you can check if its a Mobile Device by using this function:

if(isMobile.any()) {
   alert("This is a Mobile Device");
}

So if the element is for example "#my-element":

if(isMobile.any()) {
    //either
    $("#my-element").remove()
    // or 
    $("#my-element").css("display", "none")
}

BTW, if the class you add works but you didn't know how to reach this point, you can simply:

if(isMobile.any()) {
    //either
    $("#my-element").addClass("hidden-sm") //or hidden-xs, Idk
}

CSS

You can do the same with CSS using the following method:

@media only screen and (max-device-width: 480px) {
    #my-element { display: none; }
}

Where 480 px could be the value you consider it is the max. mobile device width, but this is not really precise to determine, since this may vary, and you might find a tablet really big and a laptop whose screen is really small (best approach is JS)

However, there's no single approach that's truly foolproof. The ideal bet would be to mix and match a variety of tricks as needed, to increase the chances of successfully detecting a wider range of handheld devices.
