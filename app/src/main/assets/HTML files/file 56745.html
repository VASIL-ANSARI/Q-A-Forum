This is similar to introducing ViewModel or ViewAdapter. A simple example of this is a cell displays description of any item. Say if you give the cell a User, it displays his/her full name. If you give a Mail, it displays the subject. The point is that the cell does not care what exactly given to it (a user or a mail). It only need description of every item so it needs something that help it extract description string from every single model of different types. That's the ViewModel.

Then instead of: Cell => User or Cell => News. Use: Cell => ViewModel => User or Cell => ViewModel => News. Code sample:

class ViewModel {

    private Object _item;

    public ViewModel(Object item) {
        _item = item;
    }

    public String getDescription() {
        if (_item instanceof User) {
            return ((User)_item).getFullName();
        } else if (_item instanceof Mail) {
            return ((Mail)_item).getSubject();
        }
        return "";
    }
}
