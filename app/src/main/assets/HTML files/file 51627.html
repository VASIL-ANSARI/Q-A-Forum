Data link types for mon0 (use option -y to set): IEEE802_11_RADIO (802.11 plus radiotap header)

Then the raw packet data will begin with a radiotap header, which has fields that, if present, indicate:

  • the channel (Channel, which contains the channel frequency, not the channel number, or XChannel, which contains both - the header might have both Channel and XChannel)
  • the signal strength, in dBm (Antenna signal), or the signal strength, in dB from some arbitrary reference point (dB antenna signal).

Be warned that radiotap headers are not simple to parse.

The radiotap header has a length field, which is a little-endian value, at the beginning; it indicates how long the entire header, including all the fields, is.

After the radiotap header, there will be an 802.11 header, which will contain the source and destination MAC addresses. See the IEEE 802.11 specification.
