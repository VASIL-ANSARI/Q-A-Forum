Your easiest solution is to use the more accepted way of posting content and put your inputs inside a form element.

This will require you not to use a table but you can easily achieve table appearance with CSS afterwards.

<form action="form.php" method="post" class="order">
   <label>Quantity</label> <input id="select_quantity" type="text" /> <br/>
   <label>Select</label> <select id="select_item">
        <option selected="selected">Seleziona un prodotto...</option>
    </select> <br/>
    <input id="add_btn" type="button" value="Aggiungi" />
    <button id="order_btn" value="Ordina" />
</form>

Now in your JS:

$('form.order').submit(function(e){
  e.preventDefault();
  var data=$(this).serialize();
  var url=$(this).attr('action');
  $.post(url, data, function(response){
    // do stuff with response
  });
});

The method $.fn.serialize() is only available to form or input elements, and not other containers.

You'd still need to figure out a way to add your inputs as customers added products, but best would be

$('button.add').click(function(e){
  e.preventDefault();
  var name=$(this).closest('tr').find('[name="item"]').val();
  var quantity=$(this).closest('tr').find('[name="quantity"]').val();
  $('form.order').append('<input type="hidden" name="item['+name+']" value="'+quantity+'"/>');
});

You'll need to check for duplicates etc. but this should get you a good way to it.
