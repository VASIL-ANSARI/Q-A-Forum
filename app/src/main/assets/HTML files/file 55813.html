Assuming your code is at root or in /lib, the method will run first on the client, then on the server. This is done for latency compensation and on the client is referred to as simulation. The .isServer and .isClient give you control over what environment you want your code to run in.

If you put methods in /client then they will only run on the client and vice versa if you put them in /server.

With the new ES6 imports you have specific control over what gets imported into each environment.
