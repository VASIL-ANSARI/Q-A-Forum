THanks for editing the question with sample code. Here is a solution that sets a trigger when encountering 'BIRT' in a line, and only if set, outputs a line with 'DATE' in it. Note, if you would also only want lines with 'DATE' in it directly following the 'BIRT' line, you have to modify the code a bit ;-)

Code:

#! /usr/bin/env python
from __future__ import print_function

d = """
0 @I4@ INDI
1 NAME Milo /Vettle/
2 GIVN Milo
2 SURN Vettle
2 _MARNM Vettle
1 SEX M
1 BIRT
2 DATE 23 OCT 1930
1 FAMS @F3@
0 @I5@ INDI
1 NAME Rosy /Huleknberg/
2 GIVN Rosy
2 SURN Huleknberg
2 _MARNM Vettle
1 SEX F
1 BIRT
2 DATE 15 SEP 1928
1 DEAT Y
2 DATE 10 MAR 2010
"""

trigger_found = False
token_trigger = 'BIRT'
token_grep = 'DATE'

for line in d.split('\n'):
    if token_trigger in line:
        trigger_found = True
        continue
    if trigger_found and token_grep in line:
        print(line)
        trigger_found = False

Test run with Python v2.7.11:

2 DATE 23 OCT 1930
2 DATE 15 SEP 1928

Same with python v3.

Note: It might also be more robust (given your input as olny sample), to trigger on a token like BIRT (padded with spaces, as the text given in the sample looks like there could be textfields that might have been "entered" containing the "code") as say:

2 GIVN BIRTE

which is a valid given name. That would not change the output in this sample case, but it would set the "trigger" unintendedly.
