Your best bet is to use the Synchronizer Token Pattern. This simply means generating of random "challenge" tokens that are associated with the user's current session. These challenge tokens are then inserted within the HTML forms and links associated with sensitive server-side operations.

Since you're running ASP.NET, you could use the ViewState feature. View more info here: https://www.owasp.org/index.php/Cross-Site_Request_Forgery_(CSRF)_Prevention_Cheat_Sheet
