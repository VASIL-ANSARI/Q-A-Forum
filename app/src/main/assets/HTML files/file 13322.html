What I've found so far:

Things that fire the shouldOverrideUrlLoading method will launch an external browser if:

  • the URL is not white-listed
  • some schemes (sms:, tel:, geo:, mailto:)

But, the shouldOverrideUrlLoading method is not always fired -- on Android 2.2 and previous (and maybe 2.3?) Javascript techniques to trigger a page load (such as window.open() or document.location.href =) do not fire the method, and therefore load in the WebView regardless.

Links physically clicked by the user seem to always fire the method. Using the JS event mechanism to fire a click event on a link may also work, I haven't tried it.
