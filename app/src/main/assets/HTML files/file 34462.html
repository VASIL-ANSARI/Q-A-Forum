This is the case purely because Python defines it to be the case. A negative index is treated as counting from the end of the list (or string, or tuple) instead of the start. To quote the docs, referring to my_list[i], and extended forms my_list[i:j] and my_list[i:j:k]:

If i or j is negative, the index is relative to the end of the string: len(s) + i or len(s) + j is substituted. But note that -0 is still 0.

Most third-party types implementing the sequence protocol behave the same way, but note that they are not obliged to - it is only guaranteed behavior for the built-in sequences.
