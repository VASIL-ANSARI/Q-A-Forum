First, I want to highlight that Identity Server supports for following two profiles.

  1. SAML2 SSO Web browser based profile. (SAML2 Assertions are used) More details from here
  2. WS-Federation (Passive STS) profile. (SAML Assertion)

Normally ASP.NET is by default supports for 2 profile. Therefore you are integrating using Passive STS. It seems to be, in your configuration, you have set the url to passive STS end point https://localhost:9443/passivests. If you are using SAML2 profile, Normally you need to set it to https://localhost:9443/samlsso

Therefore the configuration that you have done in WSO2IS is not valid. They are related to SAML2 SSO profile. For passive STS you can find the configurations under WS-Federation (Passive) Configuration

However there is a bug in Passive STS that Identity Server does not add audience in to the SAML Assertion.

However, You can avoid the audience validation by setting web.config <audienceUris mode="Never" />
