This is normal behaviour, because has_many and has_one have the same concept in DB, they are the relation having 0...n (zero to many) others!

But the different is meaning:

  • has_one has the meaning 0..1, otherwise, has_many is 0..n, you see n maybe equal 1
  • Moreover, has_one & has_many help rails understand what you are doing for query, for example:

Foo.first.bar

=> SELECT  "bars".* FROM "bars" WHERE "bars"."foo_id" = $1 LIMIT 1  [["foo_id", 234]]

Foo.first.bars (without limit 1)

=> SELECT  "bars".* FROM "bars" WHERE "bars"."foo_id" = $1 [["foo_id", 234]]
