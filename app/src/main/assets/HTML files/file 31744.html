I'm not sure what the backing structure is for your Outer and the SubItems collections. If they are indexable and have the same number of SubItems, than you could perhaps do something like this:

<div style="display: table;">
     @for(var j = 0; j < Outer[0].SubItems.Count; j++) {
       <div style="table-row">
          @for(var i = 0; i < Outer.Count; i++) {             
             <div style="table-cell">
                 @Html.Raw(Outer[i].SubItems[j].Property)
             </div>
          }
        </div>
     }
</div>

On a small scale, and assuming you always have at least one Outer item, this should be fine but from a performance stand-point accessing the items out of order like that is not very (memory) cache friendly.

It might be best to leave the loop structure you had originally and just redesign your HTML using CSS to position the two columns side-by-side:

<div id="container">
     @for(var j = 0; j < Outer[0].SubItems.Count; j++) {
       <div style="tablecolumn">
          @for(var i = 0; i < Outer.Count; i++) {             
             <div class="tablecell">
                 @Html.Raw(Outer[i].SubItems[j].Property)
             </div>
          }
        </div>
     }

     <!-- make sure the container layout is displayed correctly by adding a clear fix -->
     <div class="clearfix"></div>
</div>

And style the tablecolumn and tablecell in CSS to display the elements in a vertical column. I think the most important thing is to make sure the tablecell elements all have the same height so as not to have missalignments.

.tablecolumn {
     float: left;
     width: 49%;
}

.tablecell {
     height: 25px;   /* or whatever value you find appropriate */
}
