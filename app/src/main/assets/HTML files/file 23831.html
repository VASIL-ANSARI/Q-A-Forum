First: please be aware that your code is wide open to SQL Injection! You should sanitize the user input.

If you want to remember how many questions the user has already answered, you need to store that information somewhere. If your users have no accounts on the page, there is no secure way to do this, you need you store this client side â and so the user can trick you by just using a second browser.

One way of doing this is the session library of PHP (http://www.php.net/manual/en/book.session.php); whenever the user answers a question you increase his session counter by one:

session_start();
if(!isset($_SESSION['questions_answered'])) {
  $_SESSION['questions_answered'] = 1;
} else {
  $_SESSION['questions_answered']++;
}

And before you ask a question you check the counter:

session_start();
if(isset($_SESSION['questions_answered']) && $_SESSION['questions_answered'] >= 50) {
    echo 'You have already answered 50 questions';
} else {
    // ask him a question
}

To show the user his score, you should store his session id (session_id()) in the DB together with his answers. When the user clicks the check-button you query the DB for his answers, check them and show the result.

To store the session ID in the DB you should add another column, e.g. userSessionId to the table. When you insert the data, you just add the user's session ID to the query:

$query = "INSERT INTO
        radio (questionId, userSessionId, Option1, Option2, Option3)
    VALUES (
        ".$questionId.",
        ".session_id().",
        '".$Option['Option1']."',
        '".$Option['Option2']."',
        '".$Option['Option3']."')";

Additionally you should work with a question ID and store it with the answer, otherwise you won't be able to check the answer...
