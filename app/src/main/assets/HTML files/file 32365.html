Android LMK doesn't start freeing memory at some particular time, but it depends on minfree values:

 /sys/module/lowmemorykiller/parameters/minfree

For example, if cat /sys/module/lowmemorykiller/parameters/minfree has the following values:

1536,2048,4096,5120,5632,6144

Then that means these values are thresholds memory for starting killing of these particular kind of processes.

FOREGROUND_APP:1536 
VISIBLE_APP:2048
SECONDARY_SERVER:4096
HIDDEN_APP:5120
CONTENT_PROVIDER:5632
EMPTY_APP:6144

E.g., when free memory (RAM) in system is less than (6144*4/1024)= 24 MB, then LMK will start killing EMPTY_APP processes. Similarly if still memory condition don't improve it will start killing process in EMPTY_APP->CONTENT_PROVIDER->....->FOREGROUND_APP (last to be killed, this should never happen).
