You should encapsulate the behavior of both Preferences and List<Sortkey> in your own custom object. By using a Decorator, you can easily control how this functionality works. First, use a very basic implementation that only supports runtime storing of the SortKeys.

interface SortKeyManager {
  void saveSortKeys(String name, List<SortKey> sortKeys);
  List<SortKey> getSortKeys();
}

public class SimpleSortKeyManager implements SortKeyManager {
  private List<SortKey> sortKeys;

  void saveSortKeys(String name, List<SortKey> sortkeys) {
    this.sortKeys = sortKeys;
  }

  public List<SortKey> getSortKeys() {
    return sortKeys;
  }
}

Then, add a decorator that supports saving and loading to the Preferences.

Finally, inject the SortKeyManager into your JTable's constructor:

public class YourTable extends JTable {
  private final SortKeyManager manager;

  public YourTable(SortKeyManager) {
    this.manager = manager;
  }
}

This configuration will give you complete control over the way everything works and obeys the Single Responsibility Principle and the Open Closed Principle. It also no longer binds your implementation to the way Preferences works, as it's an add-on effect to using the decorator version.
