Whenever ajax fails (for eg. if there's a routing error or any error in controller's method etc) then only it comes to error block. Here your method works fine, without error, no matter book is NULL or not. You can check in controller that if book object is NULL, then return some flag and then check that variable in AJAX success. For example:

def method
  ...
  ...
  <book> object can be invalid object.
  if book.nil?
     format.json { render json: {is_data_null: true} }
  else
    respond_to do |format|
      format.json { render json: {book: {title: title}} }
    end
  end
end

And then in AJAX success block, you can check that flag.
