This is because the hash you expected (aba1..) is the md5 hash of k + m + x while the hash you got (958a..) is the md5 hash of k + m + padding + x.

The length extension attack lets you generate a hash h2 = md5(k + m + padding + x) based on only knowing the hash h1 = md5(k + m) and the length of the message l = len(k + m). However, as far as I know, it doesn't let you get rid of the padding between the messages, so you're left with some garbage in between.
