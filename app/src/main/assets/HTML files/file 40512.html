The delete corresponding to the new has to be accessible too, because the new expression does not only consist of calling the new operator.

Instead, it's like this:

  1. Call operator new.
  2. Call the constructor.
  3. If the constructor throws, call operator delete.

You can work around it by making both protected, and providing a static or free function for allocating them dynamically.

That has the advantage of only restricting dynamic allocation.
