First of all, a small note on terminology: what you have isnât a function declaration. This is a function declaration:

function fooBar() {
}

It creates a function named fooBar accessible through the variable fooBar. This is an assignment involving a named function expression:

var foo = function fooBar() {
};

The functionâs name is still fooBar, but the function is only bound to a fooBar variable inside the function itself and not outside. The fact that it does make the function accessible inside its scope without needing to refer to a variable in an outer scope, though, means that there are two reasons to name it:

  • For the ability to refer to the function inside itself regardless of code in the outer function!

    This can return whatever the outer function wants:

    function fooBar() {
    return fooBar.toString();
}

var baz = fooBar;
fooBar = 5;
baz(); // "5"

    This is always consistent:

    var fooBar = function fooBar() {
    return fooBar.toString();
};

var baz = fooBar;
fooBar = 5;
baz(); // "function fooBar() { â¦"
