First, instead of calling list.requestFocus, you may want to use nameTextField.clearFocus as that more accurately conveys what you're trying to do.

Now, I don't know all the details of this specific error, but my guess is that onEditorAction is called by TextView synchronously, inside its handling of the ACTION_DOWN or ACTION_UP event message. Calling requestFocus somewhere else may then disrupt the normal event flow for key presses. Again, that's just a guess.

What you could do is wrap your list.requestFocus call in a Runnable, and post that runnable to your event loop using Handler.post(Runnable). A common pattern is to hold a private field containing a Handler in your UI class (e.g. fragment or activity) for things like this:

Handler mHandler = new Handler();
...
mHandler.post(...);

This will let the current event/message finish processing and only execute the Runnable (in this case a call to list.requestFocus after all other currently posted messages have been processed (this is usually only a few milliseconds).

And as I said above, try using clearFocus instead.
