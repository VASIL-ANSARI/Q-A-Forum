According to MSDN ( http://msdn.microsoft.com/en-us/library/system.globalization.regioninfo.displayname(v=vs.110).aspx ), RegionInfo.DisplayName will use the language of the installed version of .NET, which invariably is the same as the language of your Windows installation.

Instead, use RegionInfo.NativeName ( http://msdn.microsoft.com/en-us/library/system.globalization.regioninfo.nativename(v=vs.110).aspx ) which is always the local name.
