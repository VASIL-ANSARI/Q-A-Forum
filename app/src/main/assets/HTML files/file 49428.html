You need a third model to associate between movies and lists, since this is a many-to-many relationship. Here are how the model relationships might look:

class List < ActiveRecord::Base
  has_many :list_movies
  has_many :movies, through: :list_movies
end

class ListMovie < ActiveRecord::Base
  belongs_to :list
  belongs_to :movie
end

class Movie < ActiveRecord::Base
  has_many :list_movies
  has_many :lists, through: :list_movies
end

And the migration to add the join table between lists and movies:

class CreateAppointments < ActiveRecord::Migration
  def change     
    create_table :list_movies do |t|
      t.belongs_to :lists, index: true
      t.belongs_to :movies, index: true
      t.timestamps null: false
    end
  end
end

With the structure properly established, you will be able to add and delete movie's from a user's list, because they are referenced as list_movies. A list_movie simply associates a particular movie to a particular list. Only the list_movie records will actually be added or deleted when making changes to the list, not an actual movie record.

If you want to remove an actual movie, and you want all instances of that movie in user's lists to also be removed, you may want to add dependent destroy to the model, i.e. , "dependent: :destroy" and same for lists:

class List < ActiveRecord::Base
  has_many :movies, through: :list_movies, dependent: :destroy
end 
class Movie < ActiveRecord::Base
  has_many :lists, through: :list_movies, dependent: :destroy
end

Then add the appropriate routes.

Finally, in your view you will be able to do something like this:

<% @list.movies.each do |movie| %>
  <h3><%= movie.title %></h3>
  <%= link_to 'Destroy', movie, method: :delete, data: { confirm: 'Are you sure?' } %>
<% end %>
