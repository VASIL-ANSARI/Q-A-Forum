The OwinMiddleware class is provided by Microsoft for easily implementing custom middleware with their implementation of an OWIN server (Katana) but OWIN is not a Microsoft only tech. By using middleware as an object, Katana can use non-Katana middleware that doesn't inherit from OwinMiddleware.

This blog post talks about middleware signatures and implementation in more detail. The most important bit of information is that the only requirement is to have a signature matching 

Func<IDictionary<string, object>, Task

You can also check out the OWIN site, and specifically the section on Application Delegates
