You should use correct session key for language (django.utils.translation.LANGUAGE_SESSION_KEY) cause exactly this key is used to take current language code from session in LocaleMiddleware for all future http requests.

Also your html forms are incorrect cause they send language code in POST data but your url + view declarations expect that language code is located directly inside url. That's different approach. Why not to use django setlang view as is? Maybe you have problems to make it work, don't you?
