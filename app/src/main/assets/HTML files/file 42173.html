EDIT: It was pointed out to me that the font-size: 0; method has some on Safari and Android. I was pointed to this article http://css-tricks.com/fighting-the-space-between-inline-block-elements/

You can also make them blocks and float them left. Here's my fiddle example.

http://jsfiddle.net/ivanjsfiddle00/a4nu8arz/6/

html

<div>
    <div></div>
    <div></div>
    <div></div>
</div>
<div id="second">
    <div></div>
    <div></div>
    <div></div>
</div>

css

div div {
    background: #000;
    height: 100px;
    width: 100px;
    display: block;
    float: left;
    }

#second {
    clear: left;
}

ORIGINAL POST: Why use javascsript to remove the white space? As I understand it when you make elements display as inline-blocks the parent element leaves room for text. Basically, what I've read is that images are displayed this way and so inline-blocks default to leaving room for caption text. Set the parent element's font-size to zero. Here's my fiddle example.

http://jsfiddle.net/ivanjsfiddle00/a4nu8arz/2/

html

<div id="first">
    <div></div>
    <div></div>
    <div></div>
</div>
<div>
    <div></div>
    <div></div>
    <div></div>
</div>

css

div div {
    background: #000;
    height: 100px;
    width: 100px;
    display: inline-block;
}
    #first {
    font-size: 0;
}
