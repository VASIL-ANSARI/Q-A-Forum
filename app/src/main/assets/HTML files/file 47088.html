Http headers are separated by colon and delimited by crlf. The header and the body are separated by an empty line.

Hence continue to read from the TcpClient/Socket until you receive an empty line. Then parse each header, look for Content-Length (to see if you should read a body too). Finally build a HttpWebRequest and send everything.

Once you receive a reply you need to read everything from the HttpWebResponse, build a valid HTTP response message and finally send everything back.
