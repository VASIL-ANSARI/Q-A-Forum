However, it does translate user input via an element other than a form field; such elements can be created using the contenteditable attribute, which is widely (though not quite universally) supported in modern browsers. In my book Going Global with JavaScript and Globalize.js, I describe some of the issues in automatic translation and suggest that although it might not be a good idea to have widgets for it on web pages in general, it could be useful for user input. For example, consider a discussion forum where discussions take place in English. You could add functionality that lets everyone use his native language, have it instantly translated into English, check it out, and submit it.
