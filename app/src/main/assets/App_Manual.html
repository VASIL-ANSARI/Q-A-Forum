Ask Q&A User Manual

About the application

Ask Q&A is a question and answer application for Android OS. This app provides a platform for software engineers and aspiring programmers to ask subjective questions related to some technology, software development or a domain of computer science whose answers may not be available through search engines as Google, Bing, etc. Any user who knows the answer to the question can answer the question posted on the app. This app also provides the facility to comment upon the question so as the users can clarify the meaning of the question by posting comments on that. On posting the answers, comments to a question, the notification will be sent to the recipient on all his devices where he is logged in ton the app.

Currently, features of commenting upon answers, editing and deletion of posted questions are not currently supported.

To ensure the validity of the question posted on the app, we use the concept of tags. A tag is predefined keyword related to a topic which a legit question must have. A question can have one or more tags. Tags make the search of the question faster for the user and also try to identify irrelevant questions. If a question cannot be assigned any of the tags defined in the app, then the question is considered as irrelevant and not allowed to post.

Any user can up vote or down vote a question, i.e., upvoting a question increases its priority to be shown first to all the users of the app and downvoting means that decreasing the priority of the question so that it is less likely to show itself to most of the users. A user can upvote or downvote a question once, i.e., if a user has already upvoted a question, then again upvoting the question will cancel his previous upvote instead and the same goes for downvote. If a user has upvoted a question previously and now attempts a downvote it, then his previous upvote is cancelled and his downvote is given to the question and vice versa.

Any user must login to the app to post questions, answers and comments and also to vote upon a question, otherwise he can see the questions and answers posted on them.

How to use the application?

The application takes few minutes after launching for initial setup. After this the app shows a list of all the questions posted on it as shown below.

  1. The Search button is used for searching the question. On clicking the button, we see the following dialog box. On selecting the search parameter(Title, Body or Tags) we get to type in the search box and the questions are displayed whose search parameter matches the entered text. If we search by tags, then we can enter tags seperated by tags to find the questions matching atleast one of the tags.

  2. The Sort button is used for changing the order of questions(if it was initially descending shown by down arrow in the button, then it becomes ascending and vice versa). It asks the parameter of ordering in th dialog box and after selecting one, it sorts the questions according to that parameter in changed order.

  3. On clicking the Add button, we are shown three buttons Tags, Users and Add Question.

    • On clicking the Tags button, we are shown various tags available in the app as in the image below.
    • On clicking the Users button, we are shown the list of users who are registered into the app. If the user is currently logged in, his status is hown as online otherwise his last login timestamp is shown.

    • On clicking the Add Question button, if the user is not logged in, a toast is shown below in the first image prompting the user to login first, otherwise it shows the dialog box to add the question. In the dialog box, we can select from Enter the text or Read from picture. In the first case we have to directly enter the body of the question while in the second case, we have to select the image and the text will be read from there.

  4. At top right, we have the Settings button. On clicking it, we are provided various options related to Sign in/out, User profile if signed in and configuration of the app.

  5. The Switch in the App bar when kept ON shows the questions posted by the user if logged in and in OFF position, it shows all the questions posted on th app. If the user is not logged in, the switch remains OFF and gets disabled so that the user may not change its position. By default, the switch remains OFF.
