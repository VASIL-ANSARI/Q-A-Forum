The message is pretty clear: you don't have an object (table, view) in your database named Login. Your query is attempting to select from that object.

Select Count(*) From Login where...

The bigger and more important issue here is that you're opening yourself up to a SQL injection attack because you're concatenating strings together to create the query. You should use parameters instead of doing that.

SqlDataAdapter sdf = new SqlDataAdapter(
    "Select Count(*) From Login where Username=@UserName and Password=@Password", con);

command.Parameters.Add("@UserName", SqlDbType.NVarChar).Value = metroTextBox1.Text;
command.Parameters.Add("@Password", SqlDbType.NVarChar).Value = metroTextBox2.Text;
