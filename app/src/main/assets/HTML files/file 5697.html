It's a struct for an arbitrarily large integer, introduced in .NET 4.

You'd use it when you want to represent integers larger than Int64/UInt64 can cope with. For example, yesterday I wrote some code to normalize System.Decimal values. System.Decimal uses a 96-bit integer to represent its mantissa, but I wanted to work with it as an integer - so I used BigInteger.

(It's possible that I could have taken another approach just using decimal, but that's a different matter...)

As another example, there was a question asked just 45 minutes ago about representing large integers to work with them for cryptography purposes. While real crypto algorithms probably use something more specialized and efficient, using BigInteger to multiply large integers together etc is a good way of showing what logically happens in crypto code.
