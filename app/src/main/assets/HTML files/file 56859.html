First of all you're mixing up the terms "method" and "function". A method is a class member function (static or not), which is what myFunction is. The functions meMethod1 and myMethod2 are not methods, they are non-member functions, or "free" functions.

Secondly, if you want the functions myMethod1 and myMethod2 to be calable only from within the MyStaticClass.cpp source file, then I recommend you put them into its own anonymous namespace:

#include "MyStaticClass.h"

// An anonymous namespace
namespace
{
    /**
     *i want to write the doc of this function but in the header
     */
    void myMethod1(){
        //do something 1
    }

    /**
     *i want to write the doc of this function but in the header
     */
    void myMethod2(){ 
        // do something 2
    }
}

void MyStaticClass::myFunction(){
    myMethod1();
    myMethod2();
}

Furthermore, if you have a class with only static member function ("methods") and no member variables, then it's no different from a namespace. So in your case your header file might look like

#pragma once

namespace MyStaticClass
{
    /**
     * i can write the doc here :D 
     */
    void myFunction();
}

The source file doesn't have to change from what I show above.
