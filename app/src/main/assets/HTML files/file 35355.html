Since you said that for each function there is a complete different set of controls, it is better to design 3 different dialog forms and use them separately.

One way is to design the dialogs in the Resource View, use single dialog box containing 3 Property Sheets. You can hide the tabs of the property pages if you want so that you have one dialog box with 3 different forms.

Also another way is that you can use Document-View-Architecture of MFC (Microsoft Foundation Classes) and create 3 different views derived from CFormView. CFormView takes in the dialog template id (designed using Resource View).

Whenever the user chooses a function from the menu, you can activate a different view. Since you are using views instead of modal dialog boxes, it will appear as if the same view is being updated. But internally, you will have 3 separate document/views handling the processing.
