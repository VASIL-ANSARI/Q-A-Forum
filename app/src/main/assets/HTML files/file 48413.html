build.gradle of mobile-websocket-example says it uses Java-WebSocket. Probably, the limitation is coming from the implementation of WebSocketClient. Start reading from WebSocketClient.run(). A fixed-size buffer is allocated and it seems WebSocketImpl.decodeFrames(ByteBuffer) assumes that frame sizes do not exceed the buffer size.

Java-WebSocket is popular but I doubt its quality. Experienced programmers can guess quality to some extent only by reading source codes without executing the binary.
