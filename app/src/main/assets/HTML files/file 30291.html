Make sure variables graphDb and others are not local variables but fields of an instance of some class Neo4jConnection. Then, create single instance of that class (singleton), run setUp() once and use that connection whenever you need access the database. How to manage that singleton, depends on your environment (do you use Spring?). Simplest way is to have a static variable referring that singleton. Read http://stackoverflow.com/questions/2832297 and other discussions marked with java+singleton tags
