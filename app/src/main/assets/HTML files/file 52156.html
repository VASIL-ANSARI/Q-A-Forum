To answer the question from the post title, the run() method from ProxyThread class is called by JVM, after the thread has been started new ProxyThread(serverSocket.accept()).start(); and it usually contains the actual work that a thread should performed (in this case, it handles whatever the server socket receives and it accepts a connection from a client). The moment when JVM calls run() method cannot be controlled by the programmer, but is after the thread has been started.

run() method is never called explicitly by the programmer.
