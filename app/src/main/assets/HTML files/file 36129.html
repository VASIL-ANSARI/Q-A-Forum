In my understanding, the same events are fired as if you would click a button without data-rel="back".

I assume you are changing a page container. Then it's here:

http://api.jquerymobile.com/pagecontainer/

The events are at least:

  • beforechange (old page)
  • beforehide (old page)
  • beforetransition (new page)
  • beforeshow (new page)
  • change (new page)

This skeleton is in the case the new page was already loaded and created. If not, there are even more events.
