Google seems to be recommending that people use HTTP schemes rather than custom schemes. You can support both simultaneously though. Try adding an additional element within your intent filter for the HTTP scheme:

<intent-filter android:label="@string/app_name">
    <action android:name="android.intent.action.VIEW" />
    <category android:name="android.intent.category.DEFAULT" />
    <category android:name="android.intent.category.BROWSABLE" />
    <data android:scheme="appindex" android:host="boraboraapp"/>
    <!-- Support HTTP too -->
    <data android:scheme="http" android:host="yourdomain.com"/>
</intent-filter>

Then change your web page tag to:

<link rel="alternate" href="android-app://com.myapp.borabora/http/yourdomain.com/foo" />
