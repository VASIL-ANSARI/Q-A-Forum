You are not going to be able to resolve this dependency, nor does it make sense to have SQLiteDatabase (an Android-specific class) appear in anything other than an Android library/module.

You can write functional JUnit tests by extending TestCase or AndroidTestCase and including this in your AndroidManifest.xml:

<instrumentation
    android:name="android.test.InstrumentationTestRunner"
    android:targetPackage="your.package.name" />

More information here: http://developer.android.com/tools/testing/testing_android.html#JUnit
