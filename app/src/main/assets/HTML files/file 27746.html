The only time that an InterruptedException is thrown is when the method is doing some sort of sleep or wait (Thread.sleep(...), Object.wait(...), etc.). In this case, the InterruptedException will only be thrown if the thread that is calling futureTask.get() has been interrupted -- not the task itself. The thread that has called get() is waiting for the task to finish and so it could be interrupted.

If you want to check to see if the task has been interrupted then you should use Thread.currentThread().isInterrupted() inside of the task code. You interrupt a task typically by calling futureTask.cancel(true). This will stop the task from running if it has not been executed yet. If it is currently running then you will need to test for the interrupt bit in your code. Something like:

// your task should do something like this if you have a while loop
while (!Thread.currentThread().isInterrupted()) {
    ...
}
