I think, you can add an enum field to your table of sentences for types (noun and verb) or add a model for types if sentence may has both of types.

After that you can select nouns and verbs separately and make a hash like { nouns: array_of_nouns, verbs: array_of_verbs } and send it to your JavaScript, where you can search in two separate arrays.

Also you can use hashes instead of arrays and group sentences by first letter to speed search up. For example if you have 'nouns' r, react, ruby, ruby-on-rails and javascript, your hash_of_nouns would be like { 'j' => [ 'javascript' ], 'r' => ['ruby', 'react', 'ruby-on-rails', 'r'] }. After first character, entered by user, you can take array, and search in it. It will be much more faster with lots of data than simple array_of_nouns.
