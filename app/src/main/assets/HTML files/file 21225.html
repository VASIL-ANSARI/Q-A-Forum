Three options

  1. Code something that connects to A & B, consolidate the infomation, connect to C and save it there
  2. Use a product that lets you connect to databases read from tables and insert into other tables, something like SQL Server Integration Services and design a flow that does the very same thing than step 1.
  3. Depending on the engine, you may be able to make dumps of A&B and load them in C, for example you could dump all data from A&B as insert statements and execute them in C (assuming the tables already exist and that there are no PK conflicts)

That should get you going.
