In Tk the root window is named '.' (dot) and its children are named as a dot delimited path of parent names. Tkinter generates the names for you using numbers. The other thing to note is that in Tk the name of a window is also a command that provides operations on that window. So the error you have here is telling you that one of your windows no longer exists as the command that manages it is gone. I suggest that your bub_id list is being modified while you iterate over it leaving you with the potential to obtain a window name that has been destroyed elsewhere. You can avoid the error using c.winfo_exists which lets you know if the window is actually existing and works even if the window has been destroyed. But really you should try to avoid making calls on destroyed windows.
