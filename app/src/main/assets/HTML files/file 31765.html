Aside from the comments about JOINs vs WHERE clause ANSI querying, I think the bigger thing you are missing is the fact that "Uitslag" and "Sets" 1-4 are not 5 columns, but 10... each one having two columns for the respective scoreA and scoreB

Uitslag has Excel columns E+F, Set 1 has G+H, Set 2 has I+J, etc. The fact that someone "prettied the headings" with the labels spanning both columns for looks is masking the answer.

SELECT 
      WEDSTRIJDFORMULIER.*, 
      INDELING.Klas,
      SUM(UITSLAG.PuntenA) as UitslagA,
      SUM(UITSLAG.PuntenB) as UitslagB,
      SUM( case when UITSLAG.Set = 1 then UITSLAG.ScoreA else 0 end ) as Set1A,
      SUM( case when UITSLAG.Set = 1 then UITSLAG.ScoreB else 0 end ) as Set1B,
      SUM( case when UITSLAG.Set = 2 then UITSLAG.ScoreA else 0 end ) as Set2A,
      SUM( case when UITSLAG.Set = 2 then UITSLAG.ScoreB else 0 end ) as Set2B,
      SUM( case when UITSLAG.Set = 3 then UITSLAG.ScoreA else 0 end ) as Set3A,
      SUM( case when UITSLAG.Set = 3 then UITSLAG.ScoreB else 0 end ) as Set3B,
      SUM( case when UITSLAG.Set = 4 then UITSLAG.ScoreA else 0 end ) as Set4A,
      SUM( case when UITSLAG.Set = 4 then UITSLAG.ScoreB else 0 end ) as Set4B
   FROM 
      WEDSTRIJDFORMULIER, 
      INDELING, 
      UITSLAG
   WHERE 
          INDELING.Team = WEDSTRIJDFORMULIER.TeamA
      AND UITSLAG.WID = WEDSTRIJDFORMULIER.WID
   GROUP BY 
      UITSLAG.WID

Notice the case/when within the SUM() where it is specifically looking for each only when it is the corresponding "SET" for respective 1-4. If not for the appropriate sum column result, just use 0, otherwise, get the score.
