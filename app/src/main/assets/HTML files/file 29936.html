This most often happens when you load the wrong binary into GDB. You should

  • Use the canonical way to invoke GDB:

    gdb out/target/product/msm8974/obj/EXECUTABLES/vold vold.core

  • Confirm that your binary out/target/product/msm8974/obj/EXECUTABLES/vold is exactly the same as /system/bin/vold on the target system that generated the core dump (modulo stripping -- if one binary is a stripped version of another, that's ok).
