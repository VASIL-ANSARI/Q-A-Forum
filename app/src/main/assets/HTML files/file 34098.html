You can use ILSpy. Open up the DLL, choose any method to analyse and select to analyse the method. A new window will open up, which shows you three categories (Uses, Used By and Overridden By). You can navigate through the hierarchy.

If there are any free tools which can show the trace at runtime... I don't know. But I am currently using ANTS Performance Profiler which shows the trace (including performance details).

And finally there is this article on codeproject: http://www.codeproject.com/Articles/15410/Creating-a-Custom-NET-Profiler It shows how to build a "Custom .NET Profiler" and saves the trace in a textfile:
