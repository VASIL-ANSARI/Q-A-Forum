Diagnosis

The problem with your second query is that you are doing GROUP BY t.id, but the other columns being selected (t.title, r.id, and r.body) are not aggregate functions. This is, in fact, an error under standard SQL, but MySQL won't complain unless you enable the server-wide ONLY_FULL_GROUP_BY setting. Instead, MySQL just gives you non-deterministic results.

Suggested solution

select t.id, t.title, r.id, r.body
    from (
            select forum_topic_id, max(id) as id
                from forum_responses
                group by forum_topic_id
        ) as latest
        inner join forum_topics t on latest.forum_topic_id = t.id
        inner join forum_responses r on latest.id = r.id
    order by r.id desc;

Beyond MySQL

With a more capable database, one would write this using window functions, also called analytic functions. You could then avoid one of those joins. On PostgreSQL, MS SQL Server, or Oracle, the query would look more like this:

select t.id, t.title, r.id, r.body
    from (
        select forum_topic_id, id, body
            from forum_responses
            where 1 = rank(id over partition by forum_topic_id order by id desc)
        ) as r
        inner join forum_topics t on t.id = r.forum_topic_id
    order by r.id desc;
