No need to use @AroundInvoke

You must follow the .persist(), .merge(), or .remove() with a .flush() this will force the exception to be thrown at that spot if one happens. Without flush, JPA closes the transaction at a later period, and that is were the exception is being thrown (out of catch range).

Then when the exception is caught, do this to get the root problem.

while(e.getCause() != null) {
   e = e.getCause();
}
System.out.println(e.getMessage());

That is where you will see the JPA message
