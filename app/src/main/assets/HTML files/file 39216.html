I ran into the same problem with jlibs so went for C10N instead which works in a very similar way. A simple example, firstly in some startup code tell C10N you want default locales to be used (you can also add your own):

C10N.configure(new C10NConfigBase()
{
    @Override
    protected void configure()
    {
        install(new DefaultC10NAnnotations());
    }
});

Then create interfaces with annotated messages:

//docs suggest this isn't needed by default but it didn't work without this annotation
@C10NMessages
public interface ExceptionText
{
    @En("That didn't work")
    @De("Das hat nicht geklappt")
    public String simpleMessage();

    //action may need translating too!
    @En("Could not perform action {0}, failure code {1}")
    @De("Aktion {0} ist nicht mÃ¶glich, Fehlercode {1}")
    public String complexMessage(String action, int code);
}

Then using the above elsewhere:

String simple = C10N.get(ExceptionText.class).simpleMessage();
String complex = C10N.get(ExceptionText.class).complexMessage("xyz", 123);

You can also get it to use resource bundles for translation text if preferred.
